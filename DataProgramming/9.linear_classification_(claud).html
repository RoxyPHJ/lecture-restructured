01

Linear Model 개요

선형 모델이란?

선형 모델은 입력 공간을 직선(또는 초평면)으로 분리하는 가설 집합이다. 학습 모델의 가장 첫 번째 선택지로 자주 사용된다.

VC 차원(Vapnik-Chervonenkis dimension): 학습 모델이 데이터를 얼마나 많이 "표현"할 수 있는지를 측정하는 지표. 선형 모델은 VC 차원이 작아 E_in → E_out 일반화가 잘 된다.

선형 모델이 "좋은 첫 번째 선택"인 이유:

  • 단순하고 해석 가능
  • 낮은 VC 차원 → 과적합 위험 낮음 → E_in에서 E_out으로 잘 일반화
  • 다양한 문제에 유연하게 적용 가능
선형 모델이 다루는 3가지 문제
#문제 유형출력 공간 𝒴알고리즘 예
1Classification{-1, +1}PLA, Pocket
2Regressionℝ (연속값)Linear Regression
3Logistic Regression[0,1] (확률)Logistic Regression

세 문제는 서로 다르지만 관련된 알고리즘으로 풀 수 있다.

비선형 분리 가능 데이터

두 가지 선형 분리 불가능 유형
(a) 노이즈형 — A few noisy points
단 2개 예제를 이동하면 선형 분리 가능 → 선형 모델 유지 + 노이즈 허용
(b) 구조적 비선형 — Nonlinearly separable
직선으로는 분리 불가 → 비선형 변환(Nonlinear Transformation) 필요
선형 모델 적용 가능성 판단
상황판단접근 방법
(a) 노이즈형 선형 모델 유지 노이즈를 허용하면서 E_in이 작은 w를 찾음 → Pocket Algorithm
(b) 구조적 비선형 선형 모델 부적합 비선형 변환 기법 필요 (나중에 다룸)
04

NP-hard & E_in 최소화

E_in 최소화 최적화 문제

노이즈 상황 (a)에서 E_in을 최소화하는 문제:

min𝒘 ∈ ℝd+1   (1/N) · Σ [ sign(𝒘ᵀ𝒙ₙ) ≠ yₙ ]
⚠ NP-hard 문제
  • 조합론적 최적화(combinatorial optimization) → 효율적인 알고리즘 없음
  • sign(·)과 [·] 모두 이산(discrete) 성질을 가짐
  • 정확히 푸는 것 자체가 NP-hard → 근사적으로 최소화
E_in vs E_out 관계

학습의 목표: 훈련 데이터로부터 일반화가 잘 되는 가설을 찾는 것

E_in
훈련 데이터 오차
(In-sample error)
→
E_out
실제 오차
(Out-of-sample error)

선형 모델의 낮은 VC 차원이 E_in ≈ E_out을 보장해준다.

05

Pocket Algorithm

개념 요약

PLA를 단순 확장한 알고리즘으로, Gallant(1990)이 제안한 Pocket algorithm with ratchet이다.

핵심 아이디어: 반복 t까지 PLA에서 만난 가중치 중 가장 좋은 𝒘를 "주머니(pocket)"에 보관하다가, 마지막에 최적의 𝒘를 최종 가설로 반환한다.
PLA 대비 단점: Line 4에서 전체 예제를 평가해 E_in(𝒘(t+1))을 계산해야 하므로 PLA보다 훨씬 느리다.
의사코드 (Pseudocode)
1:set pocket weight vector ŵ = 𝒘(0) of PLA
2:for t = 0, ..., T−1 do
3:run PLA for one update to obtain 𝒘(t+1)
4:evaluate E_in(𝒘(t+1)) /* time consuming: O(N) */
5:if 𝒘(t+1) is better than ŵ in terms of E_in
6:then set ŵ = 𝒘(t+1)
7:return ŵ
PLA vs Pocket 비교
항목PLAPocket
선형 분리 가능 데이터수렴 보장수렴 (더 느림)
선형 분리 불가능 데이터진동, 발산최선의 w 유지
반복당 비용O(1) 업데이트O(N) 평가
최종 가설마지막 w지금까지 최적 w
적합 상황노이즈 없음노이즈 있음 (실전)

손글씨 숫자 인식 예제

문제 설정
항목내용
데이터 출처US Postal Service 우편번호 데이터베이스
이미지 크기16 × 16 픽셀 (256차원 raw input)
목표각 숫자 인식
인간 E_out약 2.5% (쉽지 않은 문제)
주요 혼동 쌍{4, 9} 및 {2, 7}
해결 전략 2단계
① 다중 클래스 → 이진 분해
10개 클래스를 여러 이진 분류 문제로 분해. 예: {1, 5}만 구별하는 이진 분류기 학습
② 특징 추출 (Feature Extraction)
256 픽셀 전체 대신 중요한 특징(intensity, symmetry)만 요약하여 저차원으로 표현
특징 추출 — 숫자 1 vs 5

Observation 1 — Intensity (강도)

숫자 5는 숫자 1보다 더 많은 검정 픽셀을 차지한다. → x₁ = 평균 픽셀 강도

Observation 2 — Symmetry (대칭성)

숫자 1은 좌우 대칭, 숫자 5는 비대칭. → x₂ = 대칭성 점수

입력 벡터: 𝒙 = (x₀, x₁, x₂)
x₀ = 1 (bias), x₁ = intensity, x₂ = symmetry

알고리즘E_inE_out반복

E_in / E_out vs 반복 횟수 그래프 (로그 스케일)
