01

Learning Unknown Target Function

핵심 질문

제한된 데이터셋이 전체 목표 함수를 결정하기에 충분한 정보를 어떻게 드러낼 수 있는가?

목표 함수의 속성
  • 목표 함수 f = 학습의 목적 (learning objective)
  • f에 대한 가장 중요한 사실: 그것은 알 수 없다 (unknown)
  • 훈련 데이터 𝒟에서 f의 값은 알지만, 이것이 f를 학습했음을 의미하지는 않음
🔑
핵심 질문: 𝒟는 𝒟 밖에 있는 어떤 정보를 우리에게 알려주는가?
→ 알려준다면 ⇒ 학습이 이루어졌다
→ 알려주지 않는다면 ⇒ 학습은 불가능하다
예시: 이진 목표 함수 학습 불가능성

6개의 훈련 예제로 9비트 벡터를 입력으로 받는 이진 목표 함수 f를 학습하는 상황을 고려하자.

Case 1

f가 패턴이 대칭일 때 +1이면?
→ 새 패턴은 비대칭 → f = −1

Case 2

f가 좌상단이 흰색일 때 +1이면?
→ 새 패턴 좌상단 흰색 → f = +1

⚠️
두 가설 모두 기존 훈련 데이터와 완전히 일치한다. 어느 것이 정답인지 판별할 정보가 충분하지 않다.
해결책: 확률적 접근

f를 확실히(for sure) 알려는 시도를 포기하고, 확률적으로(in a probabilistic way) 추론한다.

💡
확률을 이용하면, 𝒟만을 사용해서 𝒟 밖에 있는 것을 추론할 수 있다.
02

Hoeffding Inequality

Bin 실험 (A 'Bin' Experiment)
  • 빨간 공과 초록 공이 (무한히 많이) 들어있는 항아리
  • ℙ[빨간 공 선택] = μ (우리에게 미지수)
  • ℙ[초록 공 선택] = 1 − μ
  • N개의 공을 독립적으로 (복원추출) 뽑음
  • 표본 내 빨간 공 비율 = μ̂
📌
μ = 항아리 전체에서 빨간 공의 실제 비율 (미지, 고정)
μ̂ = 표본에서 관측된 빨간 공의 비율 (알려짐, 확률변수)
Hoeffding 부등식 (Hoeffding Inequality)

임의의 표본 크기 N과 ε > 0에 대해:

ℙ[ |μ̂ − μ| > ε ] ≤ 2e−2ε²N
  • 상한(upper bound)은 μ에 의존하지 않음 (항아리 크기에도 무관)
  • 표본 크기 N이 커질수록, μ̂가 μ에서 ε 이상 벗어날 확률이 지수적으로 감소
  • "μ ≈ μ̂" 라는 진술은 Probably Approximately Correct (PAC)
⚡
ε를 매우 작게 (μ̂가 μ에 가깝도록) 하면 → 더 큰 N이 필요하다.
핵심 파라미터: N → N → ∞ 이면, μ ≈ μ̂ (매우 높은 확률로)

ε가 작을수록 같은 상한에 도달하기 위해 더 큰 N이 필요하다

예시: N = 1,000
ε (허용 오차)상한 2e^(−2ε²N)의미 (신뢰 수준)
ε = 0.05 ≈ 0.0067 99% 이상의 확률로 μ ∈ [μ̂−0.05, μ̂+0.05]
ε = 0.10 ≈ 4×10⁻⁹ 99.9999996%의 확률로 μ ∈ [μ̂−0.10, μ̂+0.10]
✅
1,000개 표본만으로 99% 시간에 옳다면 → 우리는 무언가를 학습한 것이다: 𝒟를 넘어서 μ에 도달했다!

Connection to Learning

Bin 모델 ↔ 학습 문제 대응
명백한 대응 (obvious)
Bin 모델학습 문제
구슬 (marble)입력점 x ∈ 𝒳
항아리 (bin)입력 공간 𝒳
표본 (sample)훈련 데이터 𝒟
덜 명백한 대응 (less obvious)
Bin 모델학습 문제
구슬 색깔h(x) ≠ f(x) 여부
μ (실제 비율)E_out(h) 밖의 오차율
μ̂ (표본 비율)E_in(h) 안의 오차율
가설 h에 의한 항아리 색칠 (Bin Coloring ≡ Hypothesis h)

특정 가설 h ∈ ℋ를 골라 각 x ∈ 𝒳에서 f와 비교한다:

  • 초록색: h(x) = f(x) — 가설이 옳음
  • 빨간색: h(x) ≠ f(x) — 가설이 틀림
μ ≡ h가 f를 근사할 때의 오차율
μ̂ ≈ 0 ⇒ h ≈ f (전체 𝒳에서)
💡
다른 h → 다른 채색 → 다른 μ
학습 알고리즘 𝒜는 가장 좋은 h (즉, g)를 선택
04

Generalizing the Hoeffding Bound

In-sample vs Out-of-sample Error
E_in(h) — 표본 내 오차
E_in(h) = (1/N) · Σ [h(xₙ) ≠ f(xₙ)]
  • μ̂에 대응
  • 확률변수 (표본에 따라 다름)
  • 알려져 있음
E_out(h) — 표본 외 오차
E_out(h) = ℙ[h(x) ≠ f(x)]
  • μ에 대응
  • 고정됨 (확률변수 아님)
  • 알 수 없음
ℙ[ |E_in(h) − E_out(h)| > ε ] ≤ 2e−2ε²N
다중 가설: Uniform Hoeffding Bound

실제 학습에서는 전체 가설 집합 ℋ = {h₁, h₂, ..., hₘ}을 탐색한다.

Union Bound을 적용하면:

ℙ[ |E_in(g) − E_out(g)| > ε ] ≤ 2Me−2ε²N
⚠️
단점:
1. 단일 가설 상한보다 M배 느슨하다
2. M이 유한한 경우에만 의미있다 (무한 M → 더 정교한 이론 필요)
05

Feasibility of Learning

학습의 궁극적 목표

E_out(g) ≈ 0 — 그러나 확률론적 분석으로 직접 얻는 것은: E_out(g) ≈ E_in(g)

학습 가능성의 두 질문
  1. E_out(g)가 E_in(g)에 충분히 가깝게 할 수 있는가? — Hoeffding 부등식으로 답 (이론적)
  2. E_in(g)를 충분히 작게 할 수 있는가? — 학습 알고리즘 실행 후 판단 (실용적)
🎯
E_in을 E_out의 대리자(proxy)로 사용한다.
E_in ≈ 0이고 E_out ≈ E_in이면 → E_out ≈ 0 → 학습 성공!
모델 복잡도와 오차의 트레이드오프
복잡도 ↑E_inE_out−E_in
ℋ 복잡도 ↑ ↓ 감소 ↑ 증가
f 복잡도 ↑ ↑ 증가 변화 없음*

*Hoeffding 상한은 f의 복잡도에 무관하나, E_in 자체가 나빠짐

⚖️
이 복잡도 트레이드오프는 학습 이론의 핵심 주제 (e.g., VC dimension)

Error Measures

오차 (Error) vs. 노이즈 (Noise)
오차 (Error)
  • 별칭: cost, objective, risk
  • 가설이 목표 함수를 얼마나 잘 근사하는지
  • 목표로부터 얼마나 멀리 있는지 정량화
  • 오차 측도의 선택이 학습 결과에 영향
노이즈 (Noise)
  • 목표 함수의 본질에 관한 것
  • 종종 출력이 입력으로 유일하게 결정되지 않음
  • 실세계 데이터에서 흔히 발생
Pointwise Error → Overall Error

이상적으로 오차 E(h, f)는 사용자 특화되어야 하지만, 실용적으로는 점별 오차(pointwise error)로 정의:

Squared Error (회귀)
e(h(x), f(x)) = (h(x) − f(x))²

연속 출력 예측에 사용

Binary Error (분류)
e(h(x), f(x)) = [h(x) ≠ f(x)]

이진 분류에 사용

전체 오차는 점별 오차의 평균:

E_in(h) = (1/N) · Σ e(h(xₙ), f(xₙ))
E_out(h) = 𝔼ₓ[e(h(x), f(x))]
오차 측도 선택: 지문 인증 예시

오차 측도는 응용 목적에 따라 달라진다. 두 가지 오류 유형:

  • False Reject: 진짜 사용자를 거부 (h=−1, f=+1)
  • False Accept: 침입자를 허용 (h=+1, f=−1)
슈퍼마켓 (할인 적용)
f=+1 (you)f=−1 (intruder)
h=+101 (false accept)
h=−110 (false reject)0

고객 불만 > 할인 손해 → false reject를 더 많이 패널티

CIA (보안)
f=+1 (you)f=−1 (intruder)
h=+101000 (false accept!)
h=−11 (false reject)0

보안 침해는 재앙 → false accept에 극도로 높은 페널티

학습 다이어그램 (오차 측도 포함)

Noisy Targets

목표가 함수가 아닌 경우

실세계 학습 데이터는 결정론적 목표 함수가 아닌 노이즈 있는 방식으로 생성되는 경우가 많다.

💳
신용카드 예시: 동일한 특징값(나이, 성별, 연봉 등)을 가진 지원자라도 신용 승인 결과가 다를 수 있다. 목표 함수가 결정론적이지 않다!
보간 vs. 회귀 (Interpolation vs. Regression)
보간 (Interpolation) — 학습 아님
yₙ = f(xₙ)
  • 결정론적 함수 f(x) ∈ ℝ 탐색
  • yₙ = f(xₙ)을 만족해야 함
  • 노이즈 없음 → 과적합 위험
회귀 (Regression) — 노이즈 추가
yₙ = f(xₙ) + ε
  • yₙ를 근사하는 확률 분포 탐색
  • ε: 노이즈 (확률 변수)
  • 실세계 데이터를 더 잘 반영
목표 '분포' (Target Distribution)

y = f(x) 대신, 출력 y를 입력 x에 영향을 받는(affected by) 확률변수로 취급한다:

P(y|x) = 목표 분포 (target distribution)
개념수식의미
Noisy target y = f(x) + (y − f(x)) 결정론적 목표 + 순수 노이즈
Deterministic target f(x) = 𝔼[y|x] noisy target의 평균
Special case P(y|x) = 0 if y≠f(x) 결정론적 = noisy의 특수 경우
P(y|x) vs P(x)의 역할 구분
P(y|x) — 목표 분포
  • 우리가 학습하려는 것
  • 입력 x에 대한 출력 y의 조건부 분포
  • = target function + noise
P(x) — 입력 분포
  • 점 x의 상대적 중요도만 정량화
  • 학습의 목적이 아님
  • 데이터 생성 메커니즘
샘플 (x, y)는 결합 분포로 생성: P(x) · P(y|x)
⚠️
P(x)·P(y|x)를 P(x,y)로 합쳐버리면 두 개념이 혼재되므로 주의! 학습에서 두 역할은 명확히 구분되어야 한다.
08

Summary

핵심 정리
01

학습은 확률적으로만 가능하다

미지의 목표 함수 f를 확실히 알 수 없으므로, 확률 도구(Hoeffding)를 통해 𝒟 밖을 추론한다.

02

Hoeffding 부등식으로 학습 가능성 Q1에 답한다

ℙ[|E_in(h)−E_out(h)| > ε] ≤ 2Me^{−2ε²N}

N을 충분히 크게 하면 E_out ≈ E_in 보장 (이론적 답).

03

E_in ≈ 0 만들기는 학습 패러다임이 답한다 (Q2)

다양한 학습 알고리즘이 E_in을 최소화한다 (실용적 답). E_in을 E_out의 proxy로 사용.

04

오차 측도 (Error Measure)

g ≈ f에서 "≈"의 의미를 정량화. E_in(h) = (1/N)Σe(h(xₙ),f(xₙ)), E_out(h) = 𝔼ₓ[e(h(x),f(x))]

05

Noisy Target

y ~ P(y|x) 대신 y = f(x). (x₁,y₁),...,(xₙ,yₙ)은 P(x,y)=P(x)P(y|x)로 생성. E_out(h) = 𝔼_{x,y}[e(h(x),y)]

06

복잡도 트레이드오프 (Preview)

모델 복잡도 ↑ → E_in ↓ 하지만 E_out−E_in ↑. 최적점 d*_vc가 존재 (VC analysis로 다룸).
