01

Decision Tree란?

slide 3
🔑 핵심 아이디어: 데이터를 분류할 때 "이 조건을 만족하는가?"를 반복해서 물어보다 보면 결론에 도달합니다. — if-then 규칙의 연속.
예시: 암 진단을 위한 의사의 판단 과정
✅ 직관적

"if-then" 규칙의 연속 → 비전문가도 결과를 이해할 수 있다

📊 분류 + 회귀 모두 가능

연속값 예측(회귀)과 카테고리 분류에 모두 사용 가능

⚡ 전처리 최소화

정규화·표준화 없이도 작동. 수치형·범주형 특성 모두 처리

⚠ 과적합 위험

너무 깊어지면 훈련 데이터에 맞춰버림 → 해결책: Random Forest

02

Decision Tree 구조

slide 4
루트 노드 (Root Node)

트리의 시작점. 전체 데이터셋에 첫 번째 질문을 던지는 곳.
→ 가장 중요한 특성이 여기에 위치

내부 노드 (Internal Node)

중간 질문 지점. "이 조건을 만족하는가?"를 판단.
→ 각 내부 노드마다 하나의 특성(feature)에 대한 조건

브랜치 (Branch)

노드에서 나오는 가지. 조건의 예/아니오 결과에 따라 분기.
→ 2개(이진 분기) 또는 그 이상

리프 노드 (Leaf Node)

트리의 끝점. 최종 답을 제공.
→ 분류: 클래스 레이블 / 회귀: 평균값

트리 구조 예시
⚠️ 깊이(Depth): 루트 → 리프까지의 질문 수. 깊을수록 복잡한 경계를 학습하지만 과적합 위험 증가!
03

분기 기준: 불순도 (Impurity)

slide 5
🔑 핵심: 좋은 분기란 나눈 후 각 그룹이 최대한 '순수'해지는 것. 순수 = 한 그룹에 같은 클래스만 존재. 불순도는 이 '섞인 정도'를 수치로 표현.
나쁜 분기 (불순도 높음)

→ 나눠도 여전히 뒤섞여 있음 / 불순도 ↑

좋은 분기 (불순도 낮음)

→ 각 그룹이 한 클래스로만 구성 / 불순도 ↓

📊 Entropy (정보이득 기반)

정보이론에서 유래. 완전 순수 = 0, 완전 혼합 = 1

🎯 Gini 불순도

계산이 더 빠름. 완전 순수 = 0, 완전 혼합 = 0.5

04

Entropy와 정보이득 (Information Gain)

slide 6
📊 Entropy 공식
H(S) = −Σₖ pₖ · log₂(pₖ)

pₖ = 클래스 k의 비율
완전 순수: H = 0 (한 클래스만 존재)
완전 혼합: H = 1 (클래스가 동일 비율)

✏ 정보이득 (Information Gain)
IG(S,A) = H(S) − Σᵥ (|Sᵥ|/|S|)·H(Sᵥ)

H(S): 분기 전 Entropy
H(Sᵥ): 분기 후 각 그룹의 Entropy
→ IG가 클수록 좋은 분기!

📋 예제: 10개 데이터 (악성 7개, 양성 3개)

분기 전 Entropy:

H(S) = −(7/10)·log₂(7/10) − (3/10)·log₂(3/10)
= −0.7·(−0.515) − 0.3·(−1.737)
= 0.360 + 0.521 = 0.881

분기 후 (특성 A 사용 시):

그룹1 (6개): H = 0.0 (순수!)
그룹2 (4개): H = 0.811

IG = 0.881 − (6/10·0.0 + 4/10·0.811)
= 0.881 − 0.324 = 0.557
⚡ Decision Tree는 각 노드에서 정보이득(IG)이 가장 큰 특성을 선택하여 분기합니다!
05

Gini 불순도 (Gini Impurity)

slide 7
💡 Gini 불순도는 Entropy보다 계산이 빠르고 비슷한 성능. scikit-learn의 DecisionTreeClassifier 기본값.
🎯 Gini 불순도 공식
Gini(S) = 1 − Σₖ pₖ²

pₖ = 클래스 k의 비율
완전 순수: Gini = 0
완전 혼합 (2클래스): Gini = 0.5

구분EntropyGini
범위0 ~ 10 ~ 0.5
계산 속도상대적으로 느림빠름
로그 사용✓ 사용✗ 미사용
결과 차이거의 동일
기본 사용처ID3, C4.5CART, sklearn
📋 예제: 악성 6개, 양성 4개

분기 전 Gini:

1 − ((6/10)² + (4/10)²) = 1 − (0.36 + 0.16) = 0.48

분기 후: 그룹1(순수) = 0.0 / 그룹2 = 1−((1/4)²+(3/4)²) = 1−(0.0625+0.5625) = 0.375

가중 평균 Gini: (6/10)·0.0 + (4/10)·0.375 = 0.15
감소량 = 0.48 − 0.15 = 0.33 (분기 효과!)
⚡ 각 노드에서 Gini 감소량이 가장 큰 특성·임계값을 선택 → Decision Tree 알고리즘의 핵심 반복 과정!
06

Decision Tree 알고리즘: 단계별 절차

slide 8
1
루트 노드에서 시작
전체 훈련 데이터를 하나의 그룹으로 시작합니다.
2
최선의 분기 특성·임계값 탐색
모든 특성, 모든 가능한 임계값을 시도 → Gini(또는 IG)가 가장 많이 줄어드는 조합 선택
3
데이터 분할
선택된 조건으로 데이터를 두 그룹(왼쪽·오른쪽 자식 노드)으로 나눕니다.
4
재귀적 반복
각 자식 노드에 대해 2~3 단계를 반복합니다. 각 노드는 독립적으로 최선의 분기를 찾습니다.
5
종료 조건 확인 (중요!)
① 노드가 완전히 순수 (Gini = 0)   ② 최소 샘플 수 이하   ③ 최대 깊이 도달 → 리프 노드로 확정
6
리프 노드 결정
분류: 해당 노드에서 가장 많은 클래스를 예측값으로 / 회귀: 해당 노드의 평균값을 예측값으로
07

Decision Tree 수치 예제

slide 9
📋 데이터셋 (6개 환자)
환자크기불규칙진단
A작음없음● 양성
B작음있음● 양성
C작음있음● 악성
D큼없음● 악성
E큼있음● 악성
F큼있음● 악성
전체 Gini = 1−((2/6)²+(4/6)²) = 0.444
① 종양 크기로 분기하면?

크기=작음 {A,B,C}: Gini = 1−((2/3)²+(1/3)²) = 0.444
크기=큼 {D,E,F}: Gini = 0.0 (순수!)
가중 Gini = (3/6)·0.444 + (3/6)·0.0 = 0.222

Gini 감소 = 0.444 − 0.222 = 0.222
② 불규칙도로 분기하면?

불규칙=없음 {A,D}: Gini = 0.5
불규칙=있음 {B,C,E,F}: Gini = 0.375
가중 Gini = (2/6)·0.5 + (4/6)·0.375 = 0.417

Gini 감소 = 0.444 − 0.417 = 0.027
✅ 종양 크기의 Gini 감소(0.222) > 불규칙도(0.027) → 루트 노드: 종양 크기로 분기! 이후 왼쪽 그룹에서 반복.
08

과적합(Overfitting)과 가지치기(Pruning)

slide 10
⚠️ Decision Tree는 제한 없이 성장하면 훈련 데이터의 노이즈까지 외워버립니다. 깊이가 깊을수록 훈련 정확도는 올라가지만, 새로운 데이터에 대한 성능은 오히려 떨어집니다.
깊이에 따른 정확도 변화

트리 깊이가 증가할수록:

• 훈련 정확도는 지속 상승

• 테스트 정확도는 깊이 4~5 이후 하락

→ 과적합(Overfitting) 현상

적당히 제한된 트리가 더 좋은 일반화 성능을 냅니다.

✂ 사전 가지치기 (Pre-pruning)

성장 중 조기 중단

• max_depth: 최대 깊이 제한
• min_samples_split: 분기에 필요한 최소 샘플 수
• min_samples_leaf: 리프의 최소 샘플 수

✂ 사후 가지치기 (Post-pruning)

완전히 키운 뒤 가지 제거

• 검증 세트 성능이 떨어지는 서브트리 제거
• Cost-Complexity Pruning (ccp_alpha)

09

Decision Tree의 한계

slide 11
⚠ 과적합에 취약

깊어질수록 훈련 데이터 노이즈까지 학습. 가지치기를 해도 여전히 불안정.

⚠ 높은 분산 (High Variance)

데이터를 조금만 바꿔도 트리 구조가 크게 달라집니다. 예측이 불안정.

⚠ 편향된 특성 선택

값의 개수가 많은 특성을 선호하는 경향 → 공평하지 않은 분기 가능성.

💡 해결책 → Random Forest: 여러 트리를 만들어 투표 / 데이터를 랜덤으로 샘플링 / 특성도 랜덤으로 선택

앙상블(Ensemble) 학습이란?

slide 12
🔑 핵심 아이디어: 한 명의 전문가보다 여러 명의 다양한 전문가가 모여 다수결로 결정하면 더 정확. 여러 개의 약한 모델(weak learner)을 합치면 강한 모델(strong learner)을 만들 수 있음.
다수결의 힘: 직관적 이해
Bagging (병렬)

독립적인 모델들을 동시에 학습 → 결과를 평균/다수결
예) Random Forest

Boosting (순차)

앞 모델이 틀린 것에 집중해 다음 모델을 학습
예) AdaBoost, XGBoost, LightGBM

11

Bagging: Bootstrap Aggregating

slide 13
개념 정의

Bootstrap: 원본 데이터에서 복원 추출로 여러 개의 서로 다른 데이터셋을 만든다.
Aggregating: 각 모델의 예측을 합쳐 최종 답을 만든다.

Bootstrap Sampling 과정
💡 왜 다양성이 중요한가? 각 트리가 같은 데이터를 보면 같은 실수를 합니다. 서로 다른 Bootstrap 샘플을 보면 서로 다른 실수를 합니다. → 다수결을 하면 각자의 실수가 상쇄되어 전체적으로 더 정확해집니다. (오차의 상관관계를 줄이는 것이 핵심!)
12

Random Forest

slide 14
Random Forest = Decision Tree + Bagging + Random Feature Selection
🎲 랜덤 1: Bootstrap Sampling

각 트리마다 다른 훈련 데이터 사용

• 원본 N개 데이터에서 복원추출로 N개 샘플
• 약 63%의 원본 데이터 포함 (나머지는 중복)
• → 트리마다 서로 다른 데이터를 봄

🎲 랜덤 2: Random Feature Selection

각 분기마다 다른 특성 후보군 사용

• 전체 d개 특성 중 √d개(분류) 또는 d/3개(회귀)만 고려
• 각 노드마다 매번 새로 뽑음
• → 특성 간 상관관계를 줄여 트리 다양성 극대화!

단순 Bagging만 하면?

강한 특성 하나가 모든 트리의 루트에 등장 → 트리들이 비슷해짐 → 다수결 효과 반감

Random Feature까지 하면?

각 트리가 완전히 다른 구조를 가짐 → 오차의 상관관계 최소화 → 다수결 효과 극대화!

⚡ 두 가지 랜덤성이 합쳐져서 단일 Decision Tree보다 훨씬 강력하고 안정적인 모델이 완성됩니다.
13

Random Forest 알고리즘: 단계별 절차

slide 15
🔧 훈련 단계 (n_estimators개 트리 반복)
1
Bootstrap 샘플 생성
원본 훈련 데이터(N개)에서 복원 추출로 N개 샘플 생성
2
Decision Tree 성장
샘플로 트리를 키우되, 각 분기마다 전체 d개 특성 중 m개(=√d)만 랜덤 선택하여 최선 분기 탐색
3
완전히 성장 (가지치기 없음)
Random Forest는 일부러 트리를 끝까지 키웁니다. 개별 트리는 과적합해도 다수결로 상쇄!
🔍 예측 단계
4
각 트리에 새 데이터 통과
n_estimators개의 트리 각각이 독립적으로 예측값을 냄
5
결과 집계 (Aggregation)
분류: 다수결 (가장 많은 표를 받은 클래스) / 회귀: 평균값
💡 OOB(Out-of-Bag) 검증: Bootstrap에서 제외된 약 37% 데이터로 별도 검증 가능 → 추가 검증 세트 없이 성능 평가!

Feature Importance: 어떤 특성이 중요한가?

slide 16
📊 Random Forest는 학습 과정에서 각 특성이 Gini 감소에 얼마나 기여했는지 자동으로 집계함. 이를 통해 별도의 Feature Selection 과정 없이 중요한 특성을 파악할 수 있음.
계산 방법

① 각 트리에서, 특성 j를 사용한 모든 분기에서 Gini 감소량을 합산

② 전체 n_estimators개 트리에서 평균

③ 모든 특성의 중요도 합 = 1이 되도록 정규화

→ 값이 클수록 그 특성이 분류에 더 많이 기여

활용설명
중요하지 않은 특성 제거중요도가 낮은 특성(혈액형, 병원 요일)을 제거 → 모델 단순화·속도 향상
도메인 지식 검증'종양 크기'가 가장 중요하다 → 의학적으로도 맞는 결과인지 확인 가능
PCA와의 차이PCA Loading은 선형 조합 기여도, RF 중요도는 분류 기여도 → 목적에 따라 선택
⚠️ 주의: 특성들이 서로 상관관계가 높으면 중요도가 분산될 수 있습니다.

max_features각 분기에서 고려할 특성 수 (√d 또는 d/3)'sqrt'
max_depth각 트리의 최대 깊이. RF는 보통 NoneNone
oob_scoreOOB 샘플로 일반화 성능 자동 추정False
🎯 Hyperparameter 튜닝 전략

• n_estimators 먼저: 성능이 필요할 시 늘리는 방향으로, 학습이 너무 오래 걸리면 줄이는 방향으로.

• max_features 조정: 분류: √d, d/2, d 시도. 특성 간 상관관계가 높으면 √d가 효과적.

• max_depth는 RF에선 보통 None: 개별 트리의 과적합은 앙상블이 상쇄하므로, 깊이 제한이 오히려 성능을 낮출 수 있음.

응용 사례 & Random Forest의 한계

slide 20–21
🏥 의료 진단

암 양성/악성 분류, 당뇨 위험도 예측
DT: 의사에게 판단 근거 설명 가능
RF: 더 높은 정확도로 진단 보조 시스템
병원 EMR 시스템, IBM Watson Health

💳 금융 신용 평가

대출 승인/거절, 신용카드 사기 탐지
DT: 규제 기관에 판단 근거 제출 필요
RF: 수천 개의 특성으로 사기 패턴 탐지
은행 여신 시스템, PayPal 사기 탐지

🌱 환경/농업

토양 데이터로 작물 수확량 예측
날씨 데이터로 홍수/가뭄 위험 분류
RF: 다양한 센서 데이터 통합 처리
스마트팜, 기상청 예측 모델

🛒 추천 시스템

사용자 행동 데이터로 구매 예측
이탈 고객 예측 (Churn Prediction)
RF Feature Importance로 핵심 이탈 요인 파악
이커머스, 통신사 고객 관리

⚠ Random Forest의 한계와 주의사항
예측 속도 느림

100~500개의 트리를 모두 통과해야 예측. 실시간 서비스에서 응답 시간 문제.
→ 대안: 트리 수 줄이거나 LightGBM 사용

메모리 사용 많음

n_estimators개의 완전한 트리를 저장. 데이터가 크면 RAM 부족.
→ 모바일·임베디드 환경에서는 단일 DT가 더 적합

해석 어려움

100개 트리의 평균이라 "왜 이 예측이 나왔는가"를 설명하기 어려움. 규제 기관·의료 분야에서 문제.
→ 대안: SHAP 값으로 개별 예측 설명

외삽(Extrapolation) 불가

트리 기반 모델은 훈련 데이터 범위 밖의 값을 예측 못함. 주가·온도처럼 범위를 벗어난 값에 부적합.
→ 대안: 선형 회귀, 신경망

💡 Random Forest는 정형 데이터(tabular data)에서 가장 강력한 방법 중 하나입니다. 이미지·텍스트·시계열에서는 딥러닝이 더 적합합니다.
🏆 Kaggle 대회에서도 Random Forest는 항상 상위권에 드는 강력한 기준 모델(Baseline)입니다.

IG = H(S) − Σ(|Sᵥ|/|S|)·H(Sᵥ)
분기 기준 선택
IG 또는 Gini 감소량이 가장 큰 특성·임계값 선택
종료 조건 (3가지)
① Gini=0 ② 최소 샘플 이하 ③ 최대 깊이 도달
Bagging
Bootstrap(복원추출) + Aggregating(다수결/평균)
Random Forest = DT + Bagging + ?
Random Feature Selection (각 분기마다 √d개 특성만 고려)
Bootstrap 포함 비율
약 63% 원본 포함, 약 37%는 OOB(검증 활용)
Feature Importance 계산
각 트리의 Gini 감소량 합산 → 전체 평균 → 정규화 (합=1)
DT 권장 상황
설명 가능성이 중요할 때, 빠른 프로토타이핑
