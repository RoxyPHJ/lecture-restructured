01

Why SVM? — 최선의 결정 경계 찾기

Slide 3
핵심 문제
  • 로지스틱 회귀, Decision Tree도 결정 경계를 그릴 수 있습니다.
  • 두 클래스 사이에 경계선이 무수히 많을 수 있습니다.
  • → 어떤 경계가 가장 안전한가?
⚡ SVM의 답: 두 클래스로부터 가장 멀리 떨어진 결정 경계를 선택한다 — 즉, 마진(Margin)을 최대화하는 경계
직관

무수히 많은 가능한 경계선 중, SVM은 두 클래스 군집 사이의 "도로 폭"을 가장 넓게 만드는 선 하나를 수학적으로 선택합니다.

마진이 넓을수록 미래의 새 데이터가 경계 근처에서 오분류될 위험이 줄어듭니다.

02

마진(Margin)과 Support Vector

Slide 4–5
마진(Margin)이란?
  • 결정 경계 = 도로 중앙선
  • 마진 = 도로 폭 (양 차선까지의 거리 합)
  • SVM은 이 도로 폭을 최대화
  • 마진이 클수록 일반화 성능 ↑
  • 마진이 클수록 훈련 데이터 변화에 강건(Robust)
📐 마진 = 각 클래스에서 결정 경계에 가장 가까운 점까지의 거리 합
Support Vector란?
  • 마진 경계선(margin boundary)에 닿는 데이터 포인트들
  • 이 점들만이 결정 경계를 완전히 결정
  • 다른 데이터를 아무리 바꿔도 SV가 같으면 경계는 불변
  • 전체 데이터 중 소수만이 Support Vector
💡 SVM은 Support Vector들의 좌표만으로 결정 경계를 완전히 정의 → 메모리 효율적, 고차원 강력
03

마진 최대화: 수식으로 이해하기

Slide 6
결정 경계 & 마진 폭 w·x + b = 0   |   마진 폭 = 2 / ‖w‖
목표: 마진 최대화

‖w‖를 최소화 = 마진 최대화

minimize   ½‖w‖²
제약 조건

각 점이 마진 밖에 있어야 함

yᵢ (w·xᵢ + b) ≥ 1
수학적 해법
  • 이차계획법 (QP)으로 풀 수 있음
  • 실전: scikit-learn이 자동 해결
🎯 핵심만 기억: SVM은 마진을 최대로 하는 w와 b를 수학적으로 찾는 알고리즘입니다.
04

Hard Margin의 한계

Slide 7
⚠️ Hard Margin = 오류를 전혀 허용하지 않는 SVM
문제 1: 노이즈에 취약
  • 이상값 1개만 있어도 마진이 0으로 수렴
  • → 선형 분리 자체가 불가능해짐
문제 2: 과적합 위험
  • 훈련 데이터 노이즈까지 완벽히 맞추려 함
  • → 복잡한 경계 → 새 데이터 일반화 실패
💡 해결책: 약간의 오류를 허용하는 Soft Margin SVM → C 파라미터로 조절
05

Soft Margin과 C 파라미터

Slide 8
📌 Soft Margin = 마진 안쪽 오류(슬랙 변수 ξ)를 허용. C 파라미터로 오류 허용 정도 조절
Soft Margin 최적화 목적함수 Minimize   0.5 · ‖w‖² + C · Σξ
C 작을 때 (C → 0)
  • 오류를 많이 허용
  • 마진이 넓어짐
  • 단순한 결정 경계
  • → 일반화 성능 ↑
  • → 과소적합 위험
C 클 때 (C → ∞)
  • 오류를 거의 허용 안 함
  • 마진이 좁아짐
  • 복잡한 결정 경계
  • → Hard Margin에 근접
  • → 과적합 위험
⚖️ C는 Decision Tree의 max_depth와 유사: 제약 ↑ → 훈련 정확도 ↑, 일반화 ↓

동일 데이터셋에 C를 바꿔가며 SVM 학습 (악성 50개 + 양성 50개, 노이즈 5개 포함)

  • 테스트 정확도: 95%
  • → 좋은 균형
C = 100
  • 좁은 마진
  • 오류 0개 허용
  • 훈련 정확도: 100%
  • 테스트 정확도: 84%
  • → 과적합 의심!

선형 분리 불가 — Soft Margin SVM의 한계

Slide 10
남은 문제
  • Soft Margin을 써도, SVM은 기본적으로 선형 경계(직선)만 그립니다.
  • XOR 패턴, 원형 분포 같은 데이터는 절대 선형으로 분리 불가능합니다.
  • → C를 아무리 조절해도 해결 불가
💡 해결책: 데이터를 더 높은 차원으로 변환하면 선형 분리 가능! → 커널 트릭

커널 트릭 — 변환 없이 내적만 계산

Slide 12
⚠️ 직접 변환하면 문제: 100개 특성 → 2차 조합만 해도 ~5,000차원, 1,000개면 ~500,000차원!
✗ 직접 변환 (비효율)
  1. φ(x₁) 계산 (고차원 벡터)
  2. φ(x₂) 계산 (고차원 벡터)
  3. 내적 φ(x₁)·φ(x₂) 계산

→ 차원 폭발, 계산 불가

✓ 커널 트릭 (효율적)

원본 공간에서 바로:

K(x₁, x₂) = φ(x₁)·φ(x₂)

→ 변환 없이 내적 결과만!

💡 비유

표준 정규 분포 Z의 CDF를 계산할 때 복잡한 적분 ∫e⁻ᵗ²dt를 직접 계산하지 않고 Z-표를 참조하여 계산하는 것과 같은 원리

⚡ 핵심: K(x₁, x₂) = φ(x₁)·φ(x₂) ← φ를 몰라도 K 함수만 알면 고차원에서 SVM 동작!

예측: Support Vector들의 가중 투표

Slide 13
SVM 예측 함수 f(x) = sign( Σᵢ αᵢ · yᵢ · K(SVᵢ, x) + b )
항목별 직관적 의미
αᵢ

이 SV의 중요도 — 훈련 시 자동 결정 (사용자가 직접 설정하는 값이 아님)

yᵢ

+1 (양성) 또는 −1 (음성) — 투표의 방향을 결정

K(·)

새 점이 이 SV와 얼마나 비슷한가? RBF: 가까울수록 1, 멀수록 0

sign

합산 양수 → 양성 클래스, 합산 음수 → 음성 클래스

수치 예시 계산
  • SV₁(+) α=0.8, K=0.70
  • SV₂(−) α=0.6, K=0.20
  • SV₃(+) α=0.4, K=0.40
  • b = −0.30
= 0.8×(+1)×0.70 + 0.6×(−1)×0.20
+ 0.4×(+1)×0.40 − 0.30
= +0.56 − 0.12 + 0.16 − 0.30
= +0.30 → sign = +1 ✓
🎯 α · b는 scikit-learn이 자동 학습 | 사용자는 C (마진 강도) · kernel (유사도 척도) 만 선택하면 됩니다!

커널(Kernel) 함수의 종류

Slide 14
커널 수식 특징 권장 상황
Linear K(x,z) = x·z 선형 경계만
파라미터 없음		 선형 분리 가능
텍스트·고차원 희소
Polynomial K(x,z) = (x·z + c)ᵈ d차 다항 경계
파라미터: d, c		 특성 조합이 중요
이미지 처리
RBF ★
(권장)		 K(x,z) = exp(−γ‖x−z‖²) 가장 유연
파라미터: γ		 모르면 RBF 먼저!
대부분의 비선형 분류
🔧 실전 순서: RBF 먼저 시도 → 성능 부족 시 Linear·Polynomial 비교 → GridSearchCV 탐색

RBF 커널과 gamma 파라미터

Slide 15
RBF 커널 수식 K(x, z) = exp(−γ‖x − z‖²)  |  γ(gamma): 각 데이터 포인트의 영향 반경을 결정
gamma 작을 때 (γ → 0)
  • 각 점의 영향 반경이 넓음
  • 주변 많은 점을 고려
  • → 부드럽고 단순한 경계
  • → 과소적합 위험
gamma 클 때 (γ → ∞)
  • 각 점의 영향 반경이 좁음
  • 아주 가까운 점만 고려
  • → 복잡하고 구불구불한 경계
  • → 과적합 위험
🔍 최적 탐색: GridSearchCV(cv=5)로 C, gamma 동시에 탐색!
13

커널 함수 수치 예제

Slide 16
📐 두 벡터 x₁ = (1, 2), x₂ = (3, 1)에 대한 세 가지 커널 값 계산 | ‖x₁−x₂‖² = (1−3)²+(2−1)² = 4+1 = 5
Linear 커널
K(x₁,x₂) = x₁·x₂
= 1×3 + 2×1
= 5

→ 직접 내적 / 선형 분리 가능 시 적합

Polynomial (d=2, c=1)
K(x₁,x₂) = (x₁·x₂ + 1)²
= (5 + 1)²
= 36

→ 2차 특성 조합 내포 / 변환 없이 2차 효과

RBF (γ=0.5)
K(x₁,x₂) = exp(−0.5×5)
= exp(−2.5)
≈ 0.082

→ 거리 멀수록 유사도 ↓ / 0에 가까울수록 두 점이 다름

💡 커널 값 = 고차원 변환 후의 내적(유사도). 클수록 '비슷한 점', 작을수록 '다른 점'

Python 코드 (scikit-learn)

Slide 18
⚠️ StandardScaler 필수: SVM은 특성 스케일에 매우 민감
✅ Pipeline 패턴: 전처리 + 모델을 한 번에
🔍 GridSearchCV: C, gamma 자동 최적화 
from sklearn.svm import SVC
from sklearn.preprocessing import StandardScaler
from sklearn.model_selection import GridSearchCV, train_test_split
from sklearn.pipeline import Pipeline
import numpy as np

X = np.random.random((1000, 100))
y = np.zeros(1000)
y[:500] = 1
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

# SVM은 스케일 정규화 필수!
pipe = Pipeline([
    ('scaler', StandardScaler()),
    ('svc',    SVC(kernel='rbf', C=1.0, gamma='scale'))
])
pipe.fit(X_train, y_train)
print(pipe.score(X_test, y_test))   # 테스트 정확도

# 최적 C, gamma 자동 탐색
grid = GridSearchCV(
    pipe,
    {'svc__C': [0.1, 1, 10, 100],
     'svc__gamma': [0.001, 0.01, 0.1, 1]},
    cv=5, scoring='accuracy'
)
grid.fit(X_train, y_train)
print(grid.best_params_)
print(grid.best_estimator_.score(X_test, y_test))

SVM의 한계와 주의사항

Slide 20
🐌 대규모 데이터에서 느림
  • N²~N³ 복잡도: 데이터 10만 건 이상이면 학습 매우 느림
  • → 대안: LinearSVC (선형 전용, 훨씬 빠름)
⚙️ 정규화 필수
  • 특성 스케일에 매우 민감. 정규화 없으면 성능 급락
  • → 반드시 StandardScaler 또는 MinMaxScaler 적용
🔧 파라미터 튜닝 어려움
  • C·gamma·kernel 조합이 많아 튜닝 비용 큼
  • → GridSearchCV 필수 / 초보자는 RF가 편할 수 있음
📊 확률 출력 어려움
  • 기본 SVM은 클래스 레이블만 반환
  • → SVC(probability=True) + Platt Scaling 필요 (느려짐)
💡 SVM은 정형·소규모·고차원 데이터에서 강력. 대규모 이미지·텍스트에는 딥러닝이 더 적합.

핵심 요약

① 마진 최대화
SVM = 두 클래스 사이의 마진을 가장 크게 만드는 결정 경계 탐색
② Support Vector
마진 경계에 닿는 소수의 점들만이 결정 경계를 완전히 결정
③ C 파라미터
오류 허용도. 작으면 넓은 마진(일반화↑), 크면 좁은 마진(과적합↑)
④ 커널 트릭
고차원 변환 없이 내적 K(x₁,x₂)만 계산 → 비선형 분류 가능
⑤ RBF + gamma
가장 범용적인 커널. gamma가 작으면 단순, 크면 복잡한 경계
⑥ 황금 공식
StandardScaler → SVC(rbf) → GridSearchCV(C, gamma)
🎯 최종 정리: SVM은 minimize ½‖w‖² (s.t. yᵢ(w·xᵢ+b)≥1)를 QP로 풀어 마진을 최대화하는 w, b를 찾고, 비선형 문제는 커널 트릭 K(x₁,x₂)=φ(x₁)·φ(x₂)로 고차원 변환 없이 해결. 예측은 f(x)=sign(Σαᵢyᵢ K(SVᵢ,x)+b)로 Support Vector들의 가중 투표.
