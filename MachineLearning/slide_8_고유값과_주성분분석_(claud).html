01

차원 축소의 필요성 — 차원의 저주

차원의 저주 (Curse of Dimensionality)
유클리드 거리 공식에서 모든 특성이 "똑같이 중요하게" 계산됨 → d(x,z) = √(Σᵢ (xᵢ − zᵢ)²)
변수가 늘어날수록 발생하는 3가지 핵심 문제
① 거리 집중 현상

모든 점들이 거의 같은 거리에 위치

→ '가장 가까운 이웃' 구분 불가

② 노이즈의 증가

예측과 무관한 변수가 늘어날 확률 ↑

→ 혈액형 같은 환자 ≠ 가장 가까운 이웃

③ 계산 비용 증가

거리 계산 비용 O(n·d)

→ d가 크면 예측이 매우 느려짐

특성변수예측과의 연관성
종양 크기x₁매우 높음
세포 모양 불규칙도x₂매우 높음
환자 혈액형x₃거의 없음
환자 생년월일x₄거의 없음
병원 방문 요일x₅거의 없음
해결 방향 → PCA / 차원 축소
변수가 너무 많아서 문제라면 꼭 필요한 변수들만 사용하자!
02

행렬식 (Determinant)

행렬의 열벡터들이 span하는 공간의 넓이(부피), 선형 독립의 정도를 나타냄

✓ det ≠ 0 : 열벡터들이 선형 독립
  • • 넓이 = |det(A)| = 4
  • • → 두 벡터가 선형 독립
  • • → R²를 완전히 Span!
  • • → 역행렬이 존재함!
✗ det = 0 : 열벡터들이 선형 종속
  • • 넓이 = |det(A)| = 0
  • • → 두 벡터가 선형 종속
  • • → R²를 Span 못함 (1D만)
  • • → 역행렬이 없음!
det(A) = 0 ⟺ 열벡터들이 선형 종속 ⟺ 전체 공간을 Span하지 못함 → 역행렬 없음, PCA에서 제외!

Determinant 계산 방법

2×2 행렬
det(A) = ad − bc

예시: A = [[1,2],[3,4]]

det = 1×4 − 2×3 = −2 ≠ 0 → 선형 독립
3×3 행렬 (여인수 전개)
a(ei−fh) − b(di−fg) + c(dh−eg)

예시: A = [[1,0,1],[0,1,1],[1,0,1]]

det = 1(1−0) − 0 + 1(0−1) = 0 → 선형 종속
PCA 핵심 연결: det(Σ − λI) = 0 → 이 방정식의 해 λ가 Eigenvalue!
03

고유값 (Eigenvalue)

어떤 벡터 v에 행렬 A를 곱해도 방향은 그대로이고 크기만 λ배가 되는 경우
A · v = λ · v    (v ≠ 0)
이때 λ를 행렬 A의 고유값(eigenvalue), v를 고유벡터(eigenvector)라 함
λ = 3 이면?

그 방향으로 3배 늘어남

(λ > 1 : 팽창)

λ = 0.5 이면?

그 방향으로 절반으로 줄어듦

(0 < λ < 1 : 수축)

λ = 0 이면?

그 방향으로 완전히 납작해짐 (정보 소실!)

A의 역행렬이 없음 → det(A) = 0

마치 '고무줄' 같은 방향

어떤 방향으로 잡아당기면 그 방향 그대로 늘어나거나 줄어드는 특별한 방향

그 방향이 바로 Eigenvector!

Eigenvalue(λ) 구하기 : 특성방정식

A·v = λ·v → (A − λI)·v = 0 → v ≠ 0이 되려면 역행렬이 없어야 함
→ det(A − λI) = 0
①

A − λI 를 구한다

단위행렬 I에 λ를 곱해 A에서 뺌

②

det(A − λI) = 0 으로 방정식을 세운다

Characteristic Equation (특성방정식)

③

λ에 대해 방정식을 푼다 → Eigenvalue!

n×n 행렬은 최대 n개의 고유값을 가짐

📋 예제: A = [[4,1],[2,3]] 의 Eigenvalue를 구하라
① (A − λI) = [[4−λ, 1], [2, 3−λ]]
② det(A − λI) = (4−λ)(3−λ) − 1·2 = λ² − 7λ + 10 = 0
③ (λ−5)(λ−2) = 0  →  λ₁ = 5, λ₂ = 2
04

고유벡터 (Eigenvector) 구하기

Eigenvalue λ를 구했다면 (A − λI) · v = 0 에 대입하여 v를 구함

λ₁ = 5 일 때 (A = [[4,1],[2,3]])
(A − 5I)·v = 0
[[-1, 1], [2, -2]] · [v₁, v₂]ᵀ = 0
→ v₁ = v₂
∴ v = [1, 1]ᵀ
정규화: [1/√2, 1/√2]ᵀ
λ₂ = 2 일 때 (A = [[4,1],[2,3]])
(A − 2I)·v = 0
[[2, 1], [2, 1]] · [v₁, v₂]ᵀ = 0
→ 2v₁ = −v₂
∴ v = [2, -1]ᵀ
정규화: [2/√5, -1/√5]ᵀ
Eigenvector는 방향만 중요합니다. 크기는 보통 1로 정규화(normalize)합니다.
05

핵심 성질 & PCA 연결

Eigenvalue / Eigenvector 핵심 성질

  • N×N 행렬은 최대 N개의 Eigenvalue를 가진다
  • 모든 Eigenvalue가 nonzero → 역행렬 존재 (det A ≠ 0)
  • 하나라도 0이면 → 역행렬 없음 (det A = 0)
  • 대칭행렬은 항상 실수 Eigenvalue를 가짐 → 공분산 행렬은 대칭!
  • 대칭행렬의 서로 다른 Eigenvalue의 Eigenvector → 서로 직교
  • Eigenvector는 방향만 의미, 크기는 보통 1로 정규화
  • 모든 Eigenvalue의 곱 = det(A), 합 = trace(A)

Eigenvalue에서 PCA로

1

공분산 행렬 계산

특성 간 선형 관계 파악 → 대칭 행렬!

2

Eigenvalue 분해

공분산 행렬의 λ와 v를 구함

3

Eigenvalue = 분산의 크기

큰 λ = 데이터가 많이 퍼진 방향 = 중요한 방향!

4

Eigenvector = 새 축(주성분)

데이터를 가장 잘 설명하는 방향

공분산 행렬의 Eigenvector = 주성분(Principal Component)
Eigenvalue = 그 방향의 분산!
06

PCA 개요 & 분산

PCA (Principal Component Analysis): 고차원 데이터를 가장 "중요한 방향"으로 투영하여 차원을 줄이는 기법
'중요한 방향' = 데이터의 분산(variance)이 최대인 방향
분산 최대화

데이터가 가장 많이 퍼진 방향

→ 데이터를 가장 잘 구분할 수 있는 방향!

직교 방향들

첫 번째 주성분 방향에 수직인 두 번째, 세 번째...

→ 각 PC는 서로 독립적인 정보 보유

정보 손실 최소화

차원을 줄이면서도 원래 데이터의 정보를 최대한 보존

분산(Variance) vs 공분산(Covariance)

분산 = 한 축이 담고 있는 정보의 양

공분산 = 두 축 사이의 중복 정보

Cov의 절대값이 크다

x₁↑ 이면 x₂↑ or x₂↓

→ 두 축이 거의 같은 정보

→ 하나로 합칠 수 있다!

Cov ≈ 0

x₁과 x₂가 서로 독립

→ 겹치는 정보가 없음

→ 둘 다 필요

Q1. 어떤 축을 없앨 수 있나?
분산이 매우 작은 축 → 정보가 적으므로 제거
Q2. 어떤 축들을 하나로 합칠 수 있나?
공분산이 큰 축들 → 중복 정보이므로 선형 결합으로 통합
07

공분산 행렬

공분산 행렬은 어느 축을 버리고 어느 축을 합칠지 결정하는 PCA의 입력값

공분산 행렬 Σ 구조
            x₁           x₂          x₃
Σ = [ Var(x₁)    Cov(x₁,x₂) Cov(x₁,x₃) ]
    [ Cov(x₂,x₁) Var(x₂)    Cov(x₂,x₃) ]
    [ Cov(x₃,x₁) Cov(x₃,x₂) Var(x₃)    ]
대각선

각 축의 분산 = 그 축이 담은 정보의 양

비대각선

두 축의 공분산 = 두 축 사이의 중복 정보

구체적인 예시 (키·체중·성적)
x₁ (키)x₂ (체중)x₃ (성적)
x₁ (키)70703
x₂ (체중)70802
x₃ (성적)3260
키·체중 공분산 = 70 (매우 큼!) → 하나로 통합 가능
성적은 다른 두 축과 거의 무관 (공분산 ≈ 2~3) → 독립적으로 필요
이 행렬로 PCA가 하는 일

① 정보량 파악

대각선(분산)을 보고 각 축 중요도 판단 → 분산이 작은 축: 제거 대상

② 중복 정보 파악

비대각선(공분산)이 크면 두 축이 비슷한 정보 → 새로운 축으로 통합

③ Eigenvalue 분해

Σ의 Eigenvalue = 새 축의 정보량, Eigenvector = 새 축의 방향

공분산 행렬 계산 공식: Σ = (1/N) XᵀX   (X는 평균 제거된 데이터 행렬)
08

PCA 알고리즘 & 예시

단계별 절차

1

데이터 표준화 (Standardization)

각 특성의 평균을 0으로, 표준편차를 1로 조정: x'ᵢ = (xᵢ − μ) / σ

→ 단위가 다른 특성들을 공정하게 비교하기 위해 필수!

2

공분산 행렬 계산

Σ = (1/N) XᵀX   (X는 평균 제거된 데이터 행렬)

3

Eigenvalue 분해

Σ · vᵢ = λᵢ · vᵢ 를 풀어 Eigenvalue {λ₁≥λ₂≥...≥λd}와 Eigenvector {v₁, ..., vd} 구함

4

주성분 선택

Eigenvalue가 큰 순서대로 k개의 Eigenvector 선택 → k는 원하는 최종 차원 수

5

데이터 투영 (Projection)

Z = X · W   (W: 선택된 k개 Eigenvector로 구성된 행렬)

→ 원래 d차원 데이터를 k차원으로 변환 완료!

구체적인 예시 (4개 포인트 → 1D)

① 원본 데이터
x₁x₂
21
43
65
87

평균: (5, 4)
평균 제거: (-3,-3), (-1,-1), (1,1), (3,3)

② 공분산 행렬 & Eigenvalue
Σ = [[5, 5],
     [5, 5]]

det(Σ − λI) = 0

λ₁ = 10, λ₂ = 0

v₁ = [1/√2, 1/√2]ᵀ

④ 1D 투영 결과

PC1 = v₁ = [1/√2, 1/√2]ᵀ

  • (-3,-3) → -4.24
  • (-1,-1) → -1.41
  • (1,1) → +1.41
  • (3,3) → +4.24
1D로 완벽 분리!
λ₂ = 0이므로 PC2 방향으로는 정보가 없음 → 제거해도 정보 손실 없음
분산 설명률: PC1 = 10/(10+0) = 100% → 1개의 주성분으로 모든 정보 보존!
09

분산 설명력 & 데이터 투영

분산 설명률(PC_k) = λ_k / (λ₁ + λ₂ + … + λ_d)
누적 분산 설명률 = Σᵢ₌₁ᵏ λᵢ / Σλ
주성분Eigenvalue설명률누적
PC14.550.0%50.0%
PC22.022.2%72.2%
PC31.213.3%85.5%
PC40.88.9%94.4%
PC50.55.6%100.0%
일반적으로 누적 분산 설명률이 90~95%가 되도록 k를 선택
이 예시: PC1+PC2+PC3 = 85.5% → k=3 선택 적합!

데이터 투영 (Projection)

변환 공식: Z = X · W
원래 데이터 X [N×d]  ×  투영 행렬 W [d×k]  =  변환된 데이터 Z [N×k]
예시: 500개 샘플, 100개 특성 → k=10 선택 → Z: [500×10] (90% 정보 보존!)
변환 전 (원래 공간)
  • • 특성: x₁, x₂, ..., x₁₀₀
  • • 100개의 축이 필요
  • • 거리 계산 비용이 매우 높음
  • • 시각화 불가능
  • • 노이즈 특성 포함
변환 후 (주성분 공간)
  • • 특성: PC1, PC2, ..., PC10
  • • 10개의 축만 필요
  • • 빠른 거리 계산 가능
  • • 2D/3D 시각화 가능!
  • • 노이즈 제거 효과

Feature Selection & 차원 축소 비교

PCA를 통해 각 특성이 주성분에 얼마나 기여하는지 Loading(부하량)을 분석하면 원래 특성의 중요도를 파악할 수 있음

Eigenvector의 각 원소 = 해당 특성의 기여도

절댓값이 클수록 그 특성이 PC에 많이 기여

③

Python 코드 (scikit-learn)

from sklearn.decomposition import PCA
from sklearn.preprocessing import StandardScaler
import numpy as np

# 1. 데이터 표준화
X = np.random.random((100, 20))
X[:, 10:20] = X[:, :10] * X[:, :10]
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)  # X: [N × d] 행렬

# 2. PCA 수행 (주성분 10개 선택)
pca = PCA(n_components=10)
Z = pca.fit_transform(X_scaled)  # Z: [N × 10] 행렬

# 3. 분산 설명률 확인
print(pca.explained_variance_ratio_)        # 각 PC의 분산 설명률
print(pca.explained_variance_ratio_.cumsum()) # 누적 분산 설명률

# 4. Loading 행렬 확인 (Feature Importance)

PCA의 한계와 주의사항

선형 가정 (Linear Assumption)

PCA는 데이터가 선형으로 분포한다고 가정

비선형 구조(나선형, 원형 등)를 제대로 포착하지 못함

대안: Kernel PCA, t-SNE, UMAP
해석의 어려움 (Interpretability)

주성분은 원래 특성들의 선형 조합

'PC1이 클수록 ??에 해당한다' 라고 말하기 어려움

→ 의료, 법률 등 설명이 필요한 분야에서 주의
분산 = 중요도 가정

PCA는 분산이 큰 방향 = 중요한 방향이라고 가정

Label 정보를 사용하지 않아 분류에 중요한 방향을 놓칠 수 있음

→ 지도학습 문제에는 LDA(선형 판별 분석)도 고려
스케일에 민감 (Scale Sensitivity)

특성의 단위/스케일이 다르면 Eigenvalue가 편향됨

반드시 PCA 전에 표준화(StandardScaler)를 수행!

x'ᵢ = (xᵢ − μ) / σ → 필수 전처리!

🎯 핵심 요약

  • 차원의 저주 → 거리 집중·노이즈·계산 비용 문제 → PCA로 해결
  • det(A) = 0 ⟺ 선형 종속 ⟺ 역행렬 없음 → PCA에서 제외
  • 특성방정식 det(A − λI) = 0 → Eigenvalue 계산
  • (A − λI)·v = 0 → Eigenvector 계산 (크기는 1로 정규화)
  • 공분산 행렬(대칭!) → Eigenvalue = 분산, Eigenvector = 주성분 방향
  • PCA 5단계: 표준화 → 공분산 → Eigen분해 → 주성분 선택 → 투영
  • 분산 설명률 90~95%가 되도록 k 선택
  • PCA 한계: 선형 가정, 해석 어려움, 분산≠중요도, 스케일 민감
