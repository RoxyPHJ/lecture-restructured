01

동기: Beer & Diapers

Classic Anecdote

1990년대 미국 슈퍼마켓 구매 데이터 분석에서 "금요일 저녁에 기저귀를 구매하는 고객은 맥주도 함께 구매하는 경향이 있다"는 패턴이 발견되었다는 유명한 일화.

이 사례가 보여주는 핵심은, 구매 데이터를 분석하면 사람이 직관적으로 예상하기 어려운 공동 출현 패턴(co-occurrence patterns)을 자동으로 발견할 수 있다는 점이다.

핵심 메시지: 패턴 마이닝은 인간의 직관을 넘어서는 숨겨진 연관성을 데이터에서 자동으로 발굴한다.

빈발 패턴이란?

Definition

빈발 패턴(Frequent Pattern)이란 데이터셋에서 자주 등장하는 패턴이다. 데이터 내에 숨겨진 규칙성(regularity)과 관계(relationship)를 드러낸다.

왜 빈발 패턴을 연구하는가?
연관성 (Association) 상관관계 (Correlation) 인과관계 (Causality) 순차 패턴 (Sequential) 주기성 (Periodicity)

이들은 여러 핵심 데이터 마이닝 태스크의 기반(foundation)이 된다.

빈발 패턴의 유형
유형설명예시
Frequent Itemsets함께 등장하는 아이템 집합{milk, bread}
Sequential Patterns특정 순서로 등장하는 아이템스마트폰 → TV → IoT 기기
Structured Patterns복잡한 자료구조 내 패턴서브그래프, 서브트리

03

시장 장바구니 분석

Market Basket Analysis

거래 데이터(transaction data)를 분석하여 고객 구매 행동을 이해하고, 단일 쇼핑에서 함께 자주 구매되는 아이템을 식별한다.

예시 연관 규칙
computer ⇒ antivirus software

컴퓨터를 구매하는 고객은 바이러스 백신 소프트웨어도 구매할 가능성이 높음을 나타낸다.

비즈니스 적용
  • 🛒 함께 구매되는 상품을 인접 배치 → 복합 판매 증대
  • 🗺️ 의도적으로 이격 배치 → 고객 동선 유도
  • 💡 교차 판매(cross-selling) 전략
  • 🎯 관련 상품 묶음 할인 → 전체 매출 향상
두 가지 핵심 측정 지표
📊 Support (지지도)
  • 두 아이템이 함께 나타나는 전체 거래 대비 빈도
  • 예: 전체 거래의 2%에서 {computer, antivirus} 동시 등장
  • 얼마나 자주 나타나는지 측정
🎯 Confidence (신뢰도)
  • A를 구매한 고객 중 B도 구매한 비율
  • 예: 컴퓨터 구매자의 60%가 바이러스 백신도 구매
  • 규칙이 얼마나 강한지 측정

Support & Confidence 공식

핵심 표기법 (Notation)
기호의미
ℐ = {I₁,I₂,…,Iₘ}전체 아이템 집합
D데이터베이스 트랜잭션 집합 (Task-relevant data)
T ⊆ ℐ비어있지 않은 아이템셋 (트랜잭션)
TID트랜잭션 식별자
A ⇒ B연관 규칙 (A ⊂ ℐ, B ⊂ ℐ, A≠∅, B≠∅, A∩B=∅)
Lₖ빈발 k-아이템셋의 집합
Support 공식
절대 지지도 (Absolute Support / Support Count)
support_count(A ∪ B) = A와 B를 모두 포함하는 트랜잭션 수
상대 지지도 (Relative Support)
support(A ⇒ B) = P(A ∪ B) = support_count(A ∪ B) / |D|
주의: A ∪ B는 수학적 합집합이지만, 여기서의 의미는 "A와 B가 함께 등장(co-occurrence)"이다. A∩B=∅이므로 A ∪ B = A와 B의 합 = 두 아이템이 같이 있는 트랜잭션.
Confidence 공식
신뢰도 = 조건부 확률 P(B|A)
confidence(A ⇒ B) = P(B|A) = support(A ∪ B) / support(A) = support_count(A ∪ B) / support_count(A)
계산 예제 (트랜잭션 DB, min_sup = 0.5)
TIDItems
T1milk, bread, beer
T2milk, diaper, beer, coke
T3milk, bread, diaper, beer
T4bread, diaper, coke
T5milk, bread, diaper, coke
support_count({milk,bread})
등장: T1, T3, T5 → 3

support({milk,bread})
= 3/5 = 0.6 ≥ 0.5 ✓ 빈발

support({bread,coke})
= 2/5 = 0.4 < 0.5 ✗ 비빈발
rule: {milk} ⇒ {bread}
support_count({milk}) = 4
support_count({milk,bread}) = 3
confidence = 3/4 = 0.75 (75%)
연관 규칙 마이닝 2단계 프로세스
  • 1
    빈발 아이템셋 탐색 — min_sup 이상의 support를 가진 모든 아이템셋을 탐색
  • 2
    강한 연관 규칙 생성 — 빈발 아이템셋에서 min_support + min_confidence를 모두 만족하는 규칙 생성
핵심 환원: support_count(A), support_count(B), support_count(A∪B)만 알면 A⇒B와 B⇒A 모두 계산 가능 → 연관 규칙 마이닝 문제는 빈발 아이템셋 탐색 문제로 환원된다.

조합 폭발 & 해결책

Combinational Explosion 문제

빈발 아이템셋의 모든 부분집합도 빈발하다 (Apriori Property). 따라서 길이 100의 빈발 아이템셋 하나가 있으면:

k-itemset개수계산
1-itemset100개C(100,1) = 100
2-itemset4,950개C(100,2) = 4,950
………
합계≈ 1.27 × 10³⁰2¹⁰⁰ − 1
C(100,1) + C(100,2) + ··· + C(100,100) = 2¹⁰⁰ − 1 ≈ 1.27 × 10³⁰
서브셋 수 시각화 (아이템셋 길이별 총 서브셋 수)
빈발 아이템셋 길이 vs 생성되는 서브셋 총 수 (2ⁿ − 1)
해결책: 압축된 표현
Closed Frequent Itemset
  • 아이템셋 X가 빈발하고
  • X의 모든 proper superset Y의 support_count가 X보다 작은 경우
  • 완전한 정보 보존, 중복 제거
Maximal Frequent Itemset
  • 아이템셋 X가 빈발하고
  • X의 모든 proper superset이 비빈발인 경우
  • 더 간결하지만 support count 정보 손실

Apriori 알고리즘

알고리즘 개요

R. Agrawal & R. Srikant (1994)이 제안한 Boolean 연관 규칙 분석을 위한 빈발 아이템셋 탐색 알고리즘.

이름 "Apriori"는 빈발 아이템셋의 prior knowledge(사전 지식)를 활용한다는 의미에서 유래.

전략: 반복적 레벨-와이즈 탐색 (Iterative Level-wise Search) — k-itemset을 이용하여 (k+1)-itemset 생성. 각 반복마다 전체 DB 스캔 필요.
Apriori Property (반단조성, Antimonotonicity)

빈발 아이템셋의 모든 비공 부분집합도 빈발하다.

Observation 1
  • 아이템셋 ℐ가 min_sup를 충족하지 못하면
  • P(ℐ) < min_sup → ℐ는 빈발하지 않음
Observation 2
  • 아이템 A를 ℐ에 추가하면
  • P(ℐ ∪ A) ≤ P(ℐ) — 더 적거나 같게 등장
Antimonotonicity: 어떤 집합이 테스트를 통과하지 못하면, 그 집합의 모든 superset도 같은 테스트를 통과하지 못한다.
예: {milk, bread} support=0.1 < min_sup=0.3 → 비빈발
∴ {milk, beer, bread}는 더욱 적게 등장 → 당연히 비빈발 → 가지치기(Prune) 가능!
핵심 두 단계
  • J
    Join 단계 (⋈)
    빈발 (k−1)-itemset Lₖ₋₁을 자기 자신과 조인하여 후보 k-itemset Cₖ를 생성.
    조건: 앞 (k−2)개 아이템이 동일하고, 마지막 아이템만 다른 두 itemset을 결합.
    사전순 정렬 유효 조합만 생성 중복 제거
  • P
    Prune 단계
    Cₖ의 각 후보에 대해 모든 (k−1)-subset을 확인.
    어떤 (k−1)-subset이 Lₖ₋₁에 없다면 → 해당 후보 제거 (Antimonotonicity 적용).
    비빈발 subset → superset도 비빈발
Apriori 의사코드 (슬라이드 20 원본)
// Input: D (트랜잭션 DB), min_sup (최소 지지도 임계값)
// Output: L (D에서의 빈발 아이템셋)

L₁ = find_frequent_1-itemsets(D);

for (k=2; Lₖ₋₁ ≠ ∅; k++) {
    Cₖ = apriori_gen(Lₖ₋₁);  // Join + Prune으로 후보 생성
    
    for each transaction t ∈ D {   // DB 스캔
        Cₜ = subset(Cₖ, t);          // t에 포함된 후보 찾기
        for each candidate c ∈ Cₜ
            c.count++;
    }
    Lₖ = {c ∈ Cₖ | c.count ≥ min_sup};
}
return L = ∪ₖ Lₖ;

// ── apriori_gen(Lₖ₋₁) ──
for each l₁ ∈ Lₖ₋₁
    for each l₂ ∈ Lₖ₋₁
        if (l₁[1]=l₂[1]) ∧ ... ∧ (l₁[k-2]=l₂[k-2]) ∧ (l₁[k-1] < l₂[k-1]) {
            c = l₁ ⋈ l₂;           // Join: 후보 생성
            if has_infrequent_subset(c, Lₖ₋₁)
                delete c;           // Prune: 비빈발 subset 있으면 제거
            else add c to Cₖ;
        }
return Cₖ;
