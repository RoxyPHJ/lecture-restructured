확률이 중요한 이유

실세계 데이터의 불확실성

데이터 마이닝은 대규모 데이터에서 의미 있는 패턴을 발견하는 것이 목표이지만, 실세계 데이터는 본질적으로 불확실하고 노이즈를 포함한다. 측정 오류, 불완전한 정보, 무작위 변동이 존재한다.

확률론이 제공하는 것
불확실성을 수학적으로 모델링하고 추론하는 프레임워크
활용 분야
분류/클러스터링, 추천 시스템, 이상 탐지, 예측 모델링
확률이 답하는 질문들
  • 이메일이 스팸일 확률은 얼마인가?
  • 고객이 제품을 구매할 확률은 얼마인가?
  • 관찰된 패턴이 우연히 발생할 가능성은 얼마나 되는가?
핵심: 확률은 불확실성을 정량화하여 정보에 근거한 의사결정을 가능하게 한다.

03

확률의 기본 개념

실험 · 표본공간 · 사건
  • 실험 (Experiment): 결과를 확실히 예측할 수 없는 프로세스
    예) 동전 던지기, 주사위 굴리기, 광고 클릭 여부
  • 표본공간 (Sample Space) S: 실험의 모든 가능한 결과 집합
    동전: S = {Head, Tail}  |  주사위: S = {1,2,3,4,5,6}
  • 사건 (Event): 우리가 관심 있는 결과들의 집합 (표본공간의 부분집합)
    A = {2,4,6} (짝수)  |  B = {4,5,6} (4 이상)
조건부 확률 (Conditional Probability)

사건 B가 이미 발생했을 때 사건 A가 발생할 확률

정의
P(A|B) = P(A ∩ B) / P(B), P(B) > 0
예제: 주사위에서 결과가 4 이상임을 알 때(B={4,5,6}), 짝수(A={2,4,6})일 확률
P(B) = 3/6,  P(A∩B) = {4,6} = 2/6
P(A|B) = (2/6)/(3/6) = 2/3
P(A|B) = P(A∩B)/P(B)
슬라이드 5–6/22
확률 변수 (Random Variables)

확률 실험의 결과에 수치를 할당하는 함수. 실세계 불확실성을 수학적 형태로 변환한다.

이산 확률변수 (Discrete)

유한하거나 가산 무한한 값을 가짐
확률 질량 함수 (PMF)로 기술

P(X = xᵢ) ≥ 0 ∀i
ΣP(X = xᵢ) = 1
연속 확률변수 (Continuous)

구간 내 임의의 값 가능
확률 밀도 함수 (PDF)로 기술

f(x) ≥ 0 ∀x
∫f(x)dx = 1
PMF (이산) vs PDF (연속)
독립 (Independence)
  • 두 사건 A, B가 독립: 한 사건의 발생이 다른 사건의 확률에 영향을 주지 않음
  • P(A|B) = P(A)
  • 동치: P(A∩B) = P(A)·P(B)
데이터 마이닝에서의 독립:
Naive Bayes는 클래스 레이블이 주어졌을 때 특징들이 조건부 독립이라 가정한다.
예: 이메일 분류 — "free", "win", 링크 수 등 단어 특징들이 독립이라 가정. 비현실적이지만 실제로 잘 동작한다.
독립의 의미
P(A∩B)
= P(A) · P(B)
곱으로 분해 가능

베이즈 정리

사전확률 · 우도 · 사후확률
사전확률 Prior P(X)
증거 관찰 전 사건의 확률. 사건에 대한 초기 믿음을 나타냄.
우도 Likelihood P(Y|X)
가설 X가 주어졌을 때 증거 Y를 관찰할 확률. 가설이 데이터를 얼마나 잘 설명하는지 나타냄.
사후확률 Posterior P(X|Y)
증거 관찰 후 사건의 확률. 새로운 정보를 사용해 사전확률을 업데이트.
주변확률 Marginal P(Y)
정규화 상수. P(Y) = ΣP(Y|Xᵢ)P(Xᵢ) — 모든 가능한 Xᵢ에 대한 합산.
베이즈 정리 (Bayes' Theorem)
핵심 공식
P(X|Y) = P(Y|X) · P(X) / P(Y)
P(X|Y)
사후확률
P(Y|X)
우도
P(X)
사전확률
P(Y)
주변확률 (정규화)
예제: 가방에서 흰 공 뽑기

가방 A: 흰 공 7개, 파란 공 3개
가방 B: 흰 공 9개, 파란 공 6개 (복원 추출)

  • P(A) = 1/2, P(B) = 1/2 (가방 선택 확률 동일)
  • P(white|A) = 7/10 = 0.7 (조건부 확률)
  • P(A, white) = P(white|A)·P(A) = 0.7 × 0.5 = 0.35 (결합)
  • P(white) = 0.7×0.5 + 0.6×0.5 = 0.65 (주변)
  • P(A|white) = P(white|A)·P(A)/P(white) = 0.35/0.65 = 7/13 ≈ 0.538
흰 공을 뽑았다는 증거가 주어지면, 가방 A에서 뽑혔을 사후확률이 사전확률(0.5)에서 0.538로 업데이트된다.
확률 트리 (복원 추출)

베이지안 분류기

베이지안 분류기 핵심 아이디어

베이지안 의사결정이론: 확률과 의사결정에 따른 비용을 사용한 trade-off를 정량화하는 통계적 학습 접근법.
예) 컨베이어 벨트 위의 물고기를 농어(sea bass)와 연어(salmon)로 분류

베이즈 분류기 (Posterior Classifier)
P(wᵢ|x) = p(x|wᵢ) · P(wᵢ) / p(x)
P(wᵢ|x) 사후확률
x가 클래스 wᵢ에 속할 확률 → 분류의 기준
p(x|wᵢ) 우도
클래스 wᵢ가 주어졌을 때 x를 관찰할 확률
P(wᵢ) 사전확률
x를 관찰하기 전 클래스 wᵢ의 확률
p(x) 증거
정규화 상수 — 모든 클래스에 걸쳐 동일하므로 분류 시 무시 가능
분류기 비교: MAP vs ML
핵심 통찰: 분류 시에는 클래스 간 비교만 필요하므로 분모 p(x)를 무시할 수 있다 → MAP classifier
분류기 결정 규칙 의미 조건
Posterior (Bayes) argmax p(wᵢ|x) 가장 높은 사후확률의 클래스 선택 이론적 최적
MAP Classifier argmax p(x|wᵢ)P(wᵢ) 우도 × 사전확률 사전확률 상이
ML Classifier argmax p(x|wᵢ) 우도만 사용 사전확률 동일(0.5)
관계: Posterior Classifier →베이즈 정리→ MAP Classifier →동일 사전확률→ ML Classifier

오류 확률

오류 확률 (Error Probability)

최고의 분류기라도 클래스 분포가 겹칠 때 오류는 불가피하다. 오류 확률 = 분류기가 샘플을 틀린 클래스로 할당할 확률.

오류 확률 (동일 사전확률 가정)
E = (1/2)∫₋∞ᵗ p(x|w₂)dx
+ (1/2)∫ₜ∞ p(x|w₁)dx
  • R₁, R₂: 클래스 w₁, w₂에 대한 결정 영역
  • 적분값: 잘못 분류된 확률 질량
  • p(x|w₁)과 p(x|w₂)의 겹침이 클수록 오류 증가
  • 결정 경계 t에서 두 분포의 교차점 → 최소 오류
두 클래스의 겹침 영역 = 오류 영역 (슬라이드 16)

최소 위험 베이지안 분류기

왜 최소 위험인가?

오류율 최소화는 모든 오분류가 동일한 비용을 갖는다고 암묵적으로 가정한다. 그러나 실제로는 거짓 경보(False Alarm)와 미탐지(Missed Detection)의 결과가 다를 수 있다.

예) 암 진단: "암이 아닌데 암으로 진단" vs "암인데 암이 아닌 것으로 진단" → 후자가 훨씬 위험
손실 행렬 (Loss Matrix)

손실 행렬 C는 잘못된 결정에 대한 패널티를 정량화한다.

손실 행렬 C
C = [[c₁₁, c₁₂],
[c₂₁, c₂₂]]

cᵢⱼ: 실제 클래스 wⱼ인데 wᵢ로 결정했을 때의 손실
  • c₁₁, c₂₂: 올바른 분류의 손실 (보통 0 또는 낮은 값)
  • c₁₂, c₂₁: 잘못된 분류의 손실 (보통 양수, cᵢⱼ > 0 when i ≠ j)
손실 행렬 시각화
결정: w₁ 결정: w₂
실제: w₁ c₁₁ (올바름) c₁₂ (오류!)
실제: w₂ c₂₁ (오류!) c₂₂ (올바름)
결정 규칙과 우도비 (Likelihood Ratio)
평균 위험 (Average Risk)
D = Σᵢ ∫ᴿⁱ (Σⱼ cᵢⱼ · p(x|wⱼ) · P(wⱼ)) dx
우도비 결정 규칙 (Likelihood Ratio Decision Rule)
x를 w₁으로 분류 if: p(x|w₁)/p(x|w₂) > (c₂₁ - c₂₂)P(w₂) / (c₁₂ - c₁₁)P(w₁)
핵심: 손실값이 결정 경계를 이동시킨다.
c₁₂ = c₂₁일 때 → MAP Classifier와 동일 (MAP는 최소 위험 분류기의 특수 케이스)
예) c₁₂=1, c₂₁=0.5 → 임계값이 좌측으로 이동 → w₁으로 분류되는 영역 증가

나이브 베이즈 스팸 필터

특정 단어들은 스팸에서 더 자주 등장한다.

예제 이메일

Click here to claim your reward.
  • w₁ = "free", w₂ = "discount", w₃ = "click"
  • 단어들이 클래스 레이블이 주어지면 독립이라 가정 (나이브 가정)
  • P(wᵢ|spam) — 훈련 데이터에서 추정
나이브 베이즈 공식
P(spam|w₁,...,wₙ)
∝ P(spam) · Π P(wᵢ|spam)
확률 추정 방법
P(spam) = n(spam) / n
P(Xᵢ=1|spam) = n(spam, wᵢ) / n(spam)
n (전체 이메일 수)
사전확률 추정에 사용
n(spam, wᵢ)
스팸 중 wᵢ 단어 포함 수

핵심 요약

2강 — Bayesian Decision Theory 핵심 정리
📊 베이즈 정리

P(X|Y) = P(Y|X)·P(X)/P(Y)
사전확률을 새로운 증거로 업데이트하여 사후확률 도출. 데이터 마이닝에서 불확실성 추론의 핵심 도구.

🎯 분류기 계층

Posterior → MAP → ML 순서로 특수화. 사전확률이 동일하면 MAP=ML. 증거 p(x)는 분류 시 무시 가능.

⚠️ 오류와 위험

클래스 분포 겹침 → 불가피한 오류 발생. 최소 위험 분류기: 비대칭 손실을 고려하여 결정 경계 조정. MAP는 최소위험의 특수 케이스.

조건부 독립 가정 하에 클래스별 우도 곱으로 분류. 비현실적 가정이지만 실용적으로 높은 성능. 스팸 필터의 고전적 구현.
