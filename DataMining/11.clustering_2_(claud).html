01 HIERARCHICAL METHODS 개요 Slide 2
핵심 개념

계층적 방법은 단일 평면 파티션이 아닌 계층적 분해(hierarchical decomposition)를 만든다. 거리(distance), 밀도(density), 연속성(continuity) 기반으로 구성 가능하며, 서브스페이스 클러스터링으로도 확장할 수 있다.

핵심 한계: 병합(merge) 또는 분할(split)이 한 번 이루어지면 일반적으로 취소(undo)가 불가능하다.
Agglomerative (Bottom-up)
병합형
1

각 객체를 개별 클러스터로 시작

2

가까운 클러스터들을 반복적으로 병합

3

모든 객체가 하나의 클러스터가 될 때까지

Divisive (Top-down)
분할형
1

모든 객체를 하나의 클러스터로 시작

2

클러스터를 반복적으로 작은 것으로 분할

3

정지 조건 충족 시까지

02 WHY HIERARCHICAL CLUSTERING? Slide 3
평면 분할 방법의 한계

k-means 같은 표준 분할 방법은 단일 평면 파티션(single flat partition)만 생성한다. 하지만 현실 데이터는 여러 granularity 수준에서 계층적 구조를 갖는 경우가 많다.

예시 — 조직 계층
직원 전체 → 임원, 관리자, 직원 → 직원 내부: 선임, 담당, 신입
예시 — 필기 인식
샘플 → 숫자별(0~9) → "7" 내부: 가로선 있음/없음 스타일
덴드로그램(Dendrogram)

계층적 클러스터링은 데이터를 클러스터의 트리(hierarchy)로 구성한다. 결과는 단일 클러스터링이 아니라 덴드로그램이며, 트리를 서로 다른 높이에서 "절단"하면 다양한 클러스터링을 얻을 수 있다.

03 TWO VIEWPOINTS OF HIERARCHIES Slide 4
관점 1: 표현 도구
요약·시각화

데이터에 본질적 계층 구조가 있다고 가정하지 않는다. 계층은 주로 데이터를 요약·시각화하기 위해 사용된다.

예: HR 분석에서 계층(임원→관리자→직원)을 통해 직급별 평균 연봉을 쉽게 계산
관점 2: 발견할 구조
내재 구조 복원

데이터에 내재적 계층 구조가 있다고 가정하고 이를 복원하려 한다.

예: 진화생물학 — 생물 군집화로 계통발생 트리 복원
예: 전략 게임 — 게임 구성 군집화로 전략 패턴 계층 발견
양쪽 모두에서, 계층적 표현은 다중 해상도(multiple resolutions)에서 데이터를 추론할 수 있게 한다.
04 AGGLOMERATIVE: LINKAGE MEASURES Slide 6
4가지 Linkage 공식

Ci, Cj: 두 클러스터  |  p, p': 객체  |  mi: Ci의 평균  |  ni: Ci의 객체 수

Minimum (Single Linkage)
dist_min(Ci, Cj) = min { ||p - p'|| } ← 두 클러스터 간 최소 거리
Maximum (Complete Linkage)
dist_max(Ci, Cj) = max { ||p - p'|| } ← 두 클러스터 간 최대 거리
Mean Distance
dist_mean(Ci, Cj) = ||mi - mj|| ← 두 클러스터 중심 간 거리
Average Distance
dist_avg(Ci, Cj) = (1 / ni·nj) · Σ ||p - p'|| ← 모든 쌍 평균
예제 계산

Cluster A: A1=(1,1), A2=(2,1)  /  Cluster B: B1=(5,1), B2=(6,1)

단일 = ||(2,1)-(5,1)|| = 3  |  완전 = ||(1,1)-(6,1)|| = 5  |  평균 = (9+10+11+12)/4 = ... → 4  |  중심간 = ||(1.5,1)-(5.5,1)|| = 4
Linkage 비교표
Linkage측정 방식별칭특징
Single 두 클러스터 중 최근접 객체 쌍의 거리 nearest-neighbor 국소 근접성(local proximity) 기반
Complete 두 클러스터 중 최원거리 객체 쌍의 거리 farthest-neighbor 전역 근접성(global closeness) 기반
Mean 두 클러스터 중심 간 거리 centroid linkage 계산 단순, 구형 클러스터에 적합
Average 모든 클러스터 간 객체 쌍 거리의 평균 UPGMA minimum과 maximum의 절충안
05 LINKAGE 해석 & 강점·약점 Slide 7–8
Linkage 해석
  • Single: 국소 근접성 기반 발견 → 연쇄효과(chaining) 발생 가능 (elongated cluster)
  • Complete: 전역 근접성 기반 → compact, tight 클러스터 선호 (내부 분산 크면 패널티)
  • Mean: 중심으로 요약 → 단순·효율적, 구형 클러스터에 적합
  • Average: min/max 절충 → 모든 교차 쌍 거리 활용, 더 균형적
이상치(Outlier) 민감성
Single Linkage: 이상치가 두 분리된 클러스터를 연결하는 "다리" 역할 → 원치 않는 병합
Complete Linkage: 이상치 하나가 두 클러스터 간 거리를 극단적으로 크게 만들 수 있음
Mean & Average: 극단적 최근접/최원거리 쌍의 영향을 완화 → 더 안정적
Mean vs. Average 세부 비교
Mean DistanceAverage Distance
계산 복잡도 낮음 — 클러스터 중심만 비교 높음 — 모든 쌍 거리 필요
데이터 타입 수치형(numerical)에 적합 범주형/혼합 타입에도 적용 가능 (centroid 정의 불필요)
06 WARD'S METHOD Slide 9–10
계층적 클러스터링과 분할 방법의 연결

분할 방법(예: k-means)은 SSE(Sum of Squared Errors)로 클러스터링을 평가한다. n개 점 → n개 클러스터: 각 점이 자체 클러스터 = agglomerative 시작점과 동일하다.

자연스러운 휴리스틱: 병합 시 SSE 증가가 가장 작은 쌍을 선택 → 최대한 compact 유지
Ward's Criterion (공식)
W(Ci, Cj) = SSE(Ci∪Cj) - SSE(Ci) - SSE(Cj)

= (ni·nj / (ni+nj)) · ||mi - mj||²

= cluster size × distance between centroids
해석: 두 클러스터를 병합했을 때 within-cluster SSE 증가량 = Ward의 유사도 측도
SSE 증가가 가장 작은 쌍 = 가장 유사한 쌍
특성:
· 중심이 가깝고 크기도 고려
· compact·low-variance 클러스터 유지
· 탐욕적(greedy) 전략
07 DIVISIVE: BASIC IDEAS Slide 11–12
Top-Down 방식
1

모든 객체를 하나의 클러스터(루트)로 시작

2

클러스터를 더 작은 서브클러스터로 반복 분할 (예: 최대 거리 기준)

3

정지 조건까지 재귀 반복: 각 클러스터가 객체 1개 OR 원하는 클러스터 수 도달

Divisive 설계 시 3가지 핵심 질문
① 분할 기준(Splitting Criterion)
가능한 분할을 어떻게 평가할 것인가?
② 분할 방법(Splitting Method)
실제로 클러스터를 어떻게 나눌 것인가?
③ 클러스터 선택 규칙
여러 클러스터 중 다음에 어느 것을 분할?
08 ISSUE 1: SPLITTING CRITERION Slide 13
어떤 분할이 더 좋은가?
수치형 데이터 → SSE
동일 클러스터 수 분할 중 더 작은 SSE를 가진 분할이 더 compact한 클러스터를 생성하므로 선호된다.
범주형 데이터 → Gini Index
클러스터의 순도(purity)를 평가하는 기준으로 사용된다.
중요: 분할 기준 선택은 최종 계층 구조에 강한 영향을 미친다.
09 ISSUE 2: SPLITTING METHOD Slide 14
실제 분할을 어떻게 수행할 것인가?

원칙적으로는 가능한 모든 파티션을 탐색하고 기준을 최적화하는 분할을 선택할 수 있다. 그러나 전수 탐색은 대용량 데이터에서 계산적으로 비현실적이다.

→ Bisecting k-means (실용적 해법)
k=2로 k-means를 실행하여 하나의 클러스터를 두 부분으로 분할한다. 전역 최적을 보장하지는 않지만, 훨씬 효율적이며 많은 응용에서 잘 작동한다.

3

k-means 반복으로 클러스터를 두 서브클러스터로 분할

결과: 두 서브클러스터 생성 (좌/우 또는 그룹 A/B)

10 ISSUE 3: WHICH CLUSTER TO SPLIT NEXT? Slide 15
클러스터 선택 규칙

분할 과정에서 여러 클러스터가 동시에 존재할 때, "가장 느슨한(loosest)" 클러스터를 선택한다. 즉, compact하지 않고 내부 분산이 가장 큰 클러스터이다.

Average SSE per Cluster
E(Ci) = (1/|Ci|) · Σ ||x - mi||²

mi: Ci의 중심 / |Ci|: 클러스터 크기
규칙: Average SSE가 가장 큰 클러스터를 다음 분할 대상으로 선택 → 내부 분산이 높아 추가 정제(refinement)에서 가장 많은 이득을 얻을 수 있기 때문
11 BIRCH: MOTIVATION & CLUSTERING FEATURE Slide 16–17
기존 계층적 방법의 2가지 주요 한계
① 비가역적 결정
병합/분할 결정을 취소할 수 없음 → 초기의 잘못된 결정이 낮은 품질의 클러스터 초래
② 확장성 부족
각 병합/분할 단계마다 많은 옵션을 검사해야 함 → 대용량 데이터에서 시간·공간 비용이 너무 큼
BIRCH = Balanced Iterative Reducing and Clustering using Hierarchies
초기 마이크로클러스터링(계층적) + 이후 매크로클러스터링(분할 방법)의 하이브리드 전략
Clustering Feature (CF): 컴팩트 클러스터 요약

n개의 d-차원 데이터 포인트 클러스터를 3-차원 벡터로 요약한다:

CF = <n, LS, SS>

n : 클러스터 내 포인트 수 LS : 선형합 Σxi (d-차원 벡터) SS : 제곱합 Σ||xi||² (스칼라)
모든 n개 포인트를 저장하는 대신 상수 크기의 <n, LS, SS>만 저장
유용한 통계량 (CF로부터 계산)

중심: x₀ = LS/n

반지름: R = √(SS/n - ||LS/n||²) ← 중심까지 평균 거리

직경: D = √((2n·SS - 2||LS||²) / n(n-1)) ← 평균 쌍별 거리
R과 D 모두 클러스터의 tightness(밀집도)를 측정한다.
12 CF-TREE & ADDITIVITY Slide 18
CF의 가산성(Additivity)

두 분리된 클러스터 C1, C2에 대해:

CF1 + CF2 = <n1+n2, LS1+LS2, SS1+SS2>

가산성 덕분에 원본 데이터 포인트를 다시 참조하지 않고도 효율적으로 클러스터를 병합·업데이트할 수 있다.
CF 계산 예제

클러스터 C1: (2,5), (3,2), (4,3)

n1 = 3 LS1 = (2+3+4, 5+2+3) = (9, 10) SS1 = 2²+3²+4² + 5²+2²+3² = 4+9+16+25+4+9 = 67 CF1 = <3, (9,10), 67>

클러스터 C2 (주어짐): CF2 = <3, (35,36), 857>
병합된 C3 = C1∪C2: CF3 = <3+3, (9+35, 10+36), 67+857> = <6, (44,46), 924>
핵심: CF 덧셈만으로 병합 완료 — 원본 6개 데이터 포인트를 다시 볼 필요가 없다!

14 BIRCH 효과성 & 한계 Slide 22
시간 복잡도
Time Complexity ≈ O(n) ← n: 객체 수에 대해 선형!

실험적으로 데이터 포인트 수에 대한 선형 확장성과 대용량 데이터셋에서의 우수한 클러스터링 품질이 확인되었다.

✓ 강점 (Strengths)
  • 매우 확장 가능하고 점진적(incremental)
  • 대용량·수치형 데이터, 스트리밍 데이터에 적합
  • 컴팩트 CF 요약으로 메모리 사용 대폭 절감
  • 다단계 정제 가능: 빠른 초기 클러스터링 후 선택적 개선
✗ 한계 (Limitations)
  • 각 CF-tree 노드는 제한된 수의 엔트리만 보유 → 노드가 "자연스러운" 클러스터에 반드시 대응하지 않을 수 있음
  • 반지름(R)·직경(D)으로 경계를 제어하므로 구형 클러스터를 암묵적으로 가정
  • 비구형(non-spherical)이나 고도로 늘어진(elongated) 클러스터에서 성능 저하
★ 핵심 요약

Hierarchical vs. Flat

계층적 방법은 단일 파티션이 아닌 다중 해상도 클러스터 트리를 생성. 병합/분할은 비가역적.

Agglomerative vs. Divisive

Bottom-up(싱글→전체 병합) vs Top-down(전체→싱글 분할). 두 방법 모두 정지 조건 지정 가능.

4가지 Linkage

Single(최근접) · Complete(최원거리) · Mean(중심간) · Average(전체 쌍 평균). 이상치 내성: Mean&Average > Single&Complete.

Ward's Method

병합 후 SSE 증가량 최소화 기준. W = (ni·nj)/(ni+nj) · ||mi-mj||². 크기와 중심 거리를 동시 고려.

Divisive 3 Key Issues

① 분할 기준(SSE/Gini) ② 분할 방법(Bisecting k-means) ③ 클러스터 선택(Average SSE 최대인 클러스터)

BIRCH: CF & CF-Tree

CF = <n, LS, SS>로 클러스터를 압축 요약. CF는 가산적 → 병합 시 원본 데이터 불필요. CF-Tree로 다단계 요약.

BIRCH 2 Phases

Phase1: CF-Tree 구축(마이크로클러스터링). Phase2: 리프에 k-means 등 적용(매크로클러스터링). O(n) 시간 복잡도.

BIRCH 한계

구형 클러스터 암묵적 가정 (R·D 기반 경계). 비구형·elongated 데이터에서 성능 저하. 노드가 자연 클러스터에 미대응 가능.
