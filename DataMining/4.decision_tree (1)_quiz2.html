Decision Tree에 대한 설명으로 올바르지 않은 것은?

A도메인 지식이나 파라미터 튜닝이 불필요하다.
B다차원 데이터를 처리할 수 있다.
C훈련 데이터에 절대 과적합(overfit)되지 않는다.

Q06객관식⭐⭐Slide 7/16

Decision Tree에서 새로운 인스턴스를 분류하는 과정으로 올바른 것은?

ARoot Node에서 시작 → 각 노드에서 테스트 조건 평가 → Branch 따라 이동 → Leaf Node 도달 → 레이블 출력
BLeaf Node에서 시작 → Branch 역방향 이동 → Root Node 도달 → 레이블 출력
C모든 Leaf Node를 탐색 → 가장 유사한 레이블 선택
DInternal Node를 무작위로 선택 → Leaf Node 도달 시 레이블 출력
📖 정답 및 해설 보기▼
정답: A
분류는 Root → Leaf 경로 탐색(path traversal)입니다. Root Node에서 시작해 각 Internal Node의 attribute test를 평가하고, feature 값에 따라 해당 Branch로 이동하며, 최종적으로 Leaf Node에 저장된 클래스 레이블이 예측값이 됩니다.

다음 중 ID3, C4.5, CART 세 알고리즘의 공통 특징이 아닌 것은?

AGreedy(non-backtracking) 전략을 사용한다.
B모두 사후 가지치기(post-pruning)를 수행한다.
CTop-down 재귀적 방식으로 트리를 구성한다.
D각 노드에서 순수한(pure) 분할을 목표로 한다.

ID3는 가지치기를 수행하지 않습니다. C4.5는 post-pruning을 포함하고, CART도 가지치기를 지원합니다. 세 알고리즘의 공통점은 Greedy 전략, Top-down 재귀적 구성, 순수 분할 목표입니다.

CART 알고리즘의 특징을 모두 올바르게 기술한 것은?

A다지 분할(multiway split)을 사용하며 Information Gain으로 분할한다.

Q10객관식⭐⭐Slide 10/16

연속형(continuous) 속성에 적용하는 분할 방식은?

A임계값 분할(threshold split): A ≤ split_point / A > split_point
B다지 분할(multi-way split): 각 값마다 별도 branch
C집합 분할(subset split): A ∈ S_A vs A ∉ S_A
D연속형 속성은 Decision Tree에서 처리 불가능하다.
📖 정답 및 해설 보기▼
정답: A
연속형 속성은 임계값 분할(threshold split)로 처리합니다. 특정 split_point를 기준으로 ≤ 와 > 두 개의 branch로 이진 분할합니다. 예: income ≤ 42,000 / income > 42,000. 다지 분할은 범주형, 집합 분할은 범주형의 이진 분할입니다.
Q11객관식⭐⭐⭐Slide 11/16

Entropy(엔트로피) 공식에서 pᵢ가 의미하는 것은?

A속성 A의 분할 수
BD에서 클래스 Cᵢ에 속하는 튜플의 비율 (|Cᵢ,D| / |D|)
C전체 데이터셋 D의 크기
D파티션 Dⱼ에 속하는 클래스의 수
📖 정답 및 해설 보기▼
정답: B
엔트로피 공식: Info(D) = −Σᵢ pᵢ · log₂(pᵢ)
pᵢ = |Cᵢ,D| / |D|
pᵢ는 데이터셋 D에서 클래스 Cᵢ에 속하는 튜플의 비율입니다. 예: yes=9, no=5, 전체=14이면 p(yes)=9/14, p(no)=5/14.
Q12객관식⭐⭐⭐Slide 13/16

14-tuple 데이터셋(yes=9, no=5)에서 4개 속성의 Information Gain을 계산했을 때, 루트 노드 분할 속성으로 선택되는 것은?

Aincome (Gain = 0.029 bits)
Bcredit_rating (Gain = 0.048 bits)
Cstudent (Gain = 0.151 bits)
Dage (Gain = 0.246 bits)
📖 정답 및 해설 보기▼
정답: D — age
Information Gain이 가장 높은 속성이 분할 속성으로 선택됩니다:
age(0.246) > student(0.151) > credit_rating(0.048) > income(0.029)
따라서 age가 루트 노드 속성으로 선택됩니다. middle_aged 파티션(4yes, 0no)은 entropy=0이므로 즉시 Leaf Node가 됩니다.
Q13객관식⭐⭐⭐Slide 15/16

Information Gain이 고유 식별자(product_ID)와 같이 값이 매우 많은 속성을 과도하게 선호하는 이유는?

A각 값이 모든 클래스를 포함하여 엔트로피가 최대가 되기 때문이다.
BSplit Information이 0이 되어 Gain Ratio가 무한대가 되기 때문이다.
C각 값이 하나의 튜플만 포함하여 각 파티션의 entropy가 0이 되고, 결과적으로 gain이 최대(Info(D))가 되기 때문이다.
D연속형 속성이기 때문에 임계값 분할이 적용되기 때문이다.
📖 정답 및 해설 보기▼
정답: C
고유 식별자처럼 값이 많은 속성은 각 값이 단 하나의 튜플만 포함하는 파티션을 만듭니다. 단일 튜플 파티션은 entropy = 0 (완전히 순수)이므로 조건부 엔트로피 Info_A(D) ≈ 0이 되고, Gain = Info(D) − 0 = 최대값이 됩니다. 하지만 이는 실제로 유용한 분할이 아닙니다.
Q14객관식⭐⭐⭐Slide 15/16

income 속성의 분포가 high=4, medium=6, low=4 (총 14)일 때, SplitInfo_income(D) 값에 가장 가까운 것은?

A1.557
B0.029
C0.940
D0.019
📖 정답 및 해설 보기▼
정답: A — 1.557
SplitInfo_income(D)
= −(4/14)·log₂(4/14) − (6/14)·log₂(6/14) − (4/14)·log₂(4/14)
= −0.2857·(−1.807) − 0.4286·(−1.222) − 0.2857·(−1.807)
= 0.516 + 0.524 + 0.516 = 1.557
0.029는 Gain(income), 0.940은 Info(D), 0.019는 GainRatio(income) = 0.029/1.557입니다.

Gini Impurity에 대한 설명으로 올바른 것은?

AGini = 1이면 완전히 순수한 상태를 의미한다.
B클래스 분포에 따라 무작위 레이블을 선택할 때의 오분류 확률로 해석할 수 있다.
CGini Index는 ID3 알고리즘에서 사용하는 분할 기준이다.
DGini(D) = Σᵢ pᵢ²로 계산한다.

Q22단답형⭐⭐Slide 15/16

Gain Ratio에서 Information Gain을 나누는 분모에 해당하는 값의 이름은? (영문으로 쓰시오)

📖 정답 및 해설 보기▼
정답: Split Information
GainRatio(A) = Gain(A) / SplitInfo_A(D)

Q23단답형⭐⭐⭐Slide 16/16

Information Gain 계산 공식의 빈칸을 채우시오.

// 속성 A의 Information Gain Gain(A) = − // 조건부 엔트로피 (Conditional Entropy) Info_A(D) = Σⱼ (|Dⱼ|/|D|) ×
📖 정답 및 해설 보기▼
정답: Info(D) / Info_A(D) / Info(Dⱼ)
Gain(A) = Info(D) − Info_A(D)
Info_A(D) = Σⱼ (|Dⱼ|/|D|) × Info(Dⱼ)
Info(D)는 분할 전 전체 엔트로피, Info_A(D)는 속성 A로 분할 후 가중 평균 엔트로피입니다. 그 차이가 정보 이득입니다.

// Gini Impurity Gini(D) = − Σᵢ pᵢ² where pᵢ = |Cᵢ,D| / |D| // Gain Ratio GainRatio(A) = Gain(A) / // Split Information SplitInfo_A(D) = −Σⱼ (|Dⱼ|/|D|) · log₂()

Q28서술형⭐⭐⭐Slide 8/16

ID3, C4.5, CART 세 알고리즘을 ① 분할 기준, ② 속성 타입 지원, ③ 트리 형태, ④ 가지치기 여부 4가지 항목으로 비교하여 서술하시오.

📖 모범 답안 보기▼
모범 답안
ID3 (Quinlan 1986)
① 분할 기준: Information Gain
② 속성 타입: 범주형(categorical)만 처리
③ 트리 형태: 다지(multi-way) 분할
④ 가지치기: 수행하지 않음

C4.5 (Quinlan 1993, ID3의 확장)
① 분할 기준: Gain Ratio (정보 이득을 Split Information으로 정규화)
② 속성 타입: 범주형 + 연속형(continuous) 모두 지원
③ 트리 형태: 다지 분할
④ 가지치기: Post-pruning 수행

CART (Breiman et al. 1984)
① 분할 기준: 분류 → Gini Index, 회귀 → MSE/RSS
② 속성 타입: 범주형 + 연속형 모두 지원
③ 트리 형태: 이진 트리(Binary Tree)만 생성
④ 가지치기: 수행

공통 특징: Greedy 전략(비역추적), Top-down 재귀적 구성, 순수(pure) 파티션 목표
Q29서술형⭐⭐⭐Slide 11–15/16

14-tuple 고객 데이터셋(yes=9, no=5)에서 age 속성의 Information Gain 계산 과정을 5단계로 서술하고, 왜 age가 루트 노드 속성으로 선택되는지 설명하시오.

📖 모범 답안 보기▼
모범 답안
Step 1: Info(D) 계산
Info(D) = −(9/14)·log₂(9/14) − (5/14)·log₂(5/14) = 0.940 bits

Step 2: age 파티션 구성
youth: 5튜플 (yes=2, no=3)
middle_aged: 4튜플 (yes=4, no=0)
senior: 5튜플 (yes=3, no=2)

Step 3: 각 파티션 엔트로피
Info(youth) = −(2/5)·log₂(2/5) − (3/5)·log₂(3/5) = 0.971
Info(middle_aged) = −(4/4)·log₂(4/4) = 0 (pure!)
Info(senior) = −(3/5)·log₂(3/5) − (2/5)·log₂(2/5) = 0.971

Step 4: 조건부 엔트로피
Info_age(D) = (5/14)·0.971 + (4/14)·0 + (5/14)·0.971 = 0.694 bits

Step 5: Information Gain
Gain(age) = 0.940 − 0.694 = 0.246 bits

루트 노드 선택 이유:
Gain(age)=0.246 > Gain(student)=0.151 > Gain(credit_rating)=0.048 > Gain(income)=0.029
ID3는 최대 Information Gain을 가진 속성을 선택하므로 age가 루트 노드가 됩니다. middle_aged 파티션은 entropy=0(순수)이므로 즉시 Leaf Node(yes)가 됩니다.
Q30서술형⭐⭐⭐Slide 6–15/16

Decision Tree의 Attribute Selection Measure(속성 선택 측도)로 사용되는 Entropy, Information Gain, Gain Ratio, Gini Index를 각각의 공식과 특징을 중심으로 비교 서술하고, 각 측도가 어떤 알고리즘에서 사용되는지 명시하시오.

📖 모범 답안 보기▼
모범 답안
① Entropy (엔트로피)
공식: Info(D) = −Σᵢ pᵢ·log₂(pᵢ), where pᵢ = |Cᵢ,D| / |D|
데이터셋 D의 불순도(impurity)를 측정. 값이 클수록 혼합, 0이면 완전 순수.
Information Gain 계산의 기반.

② Information Gain — ID3
공식: Gain(A) = Info(D) − Info_A(D)
속성 A로 분할 후 불확실성이 얼마나 감소하는지 측정.
단점: 값이 많은 속성(product_ID 등)에 편향됨 → 항상 최대 Gain을 가짐.

③ Gain Ratio — C4.5
공식: GainRatio(A) = Gain(A) / SplitInfo_A(D)
SplitInfo = −Σⱼ (|Dⱼ|/|D|)·log₂(|Dⱼ|/|D|) (분할 방식의 정보량)
정규화를 통해 많은 값을 가진 속성의 편향 완화. ID3보다 나은 일반화.

④ Gini Index — CART
공식: Gini(D) = 1 − Σᵢ pᵢ²
클래스 분포에 따라 무작위 레이블 선택 시 오분류 확률로 해석.
Gini=0: 완전 순수, 값이 클수록 혼합. 이진 분할에 최적화.

공통 목표: 가능한 한 순수(pure)한 파티션을 생성하는 속성을 선택.
