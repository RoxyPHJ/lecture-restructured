Linear Models Overview
Signal & 3가지 선형 모델
s = wTx = Σ wixi  (i = 0 to d)

모든 선형 모델은 동일한 signal s를 공유하고, 활성화 함수(activation)만 다르다.

모델 활성화 함수 h(x) 출력 범위 용도
Linear Classification sign(wTx) {+1, −1} 이진 결정 (하드)
Linear Regression wTx (−∞, +∞) 연속값 예측
Logistic Regression θ(wTx) (0, 1) 확률 예측 (소프트)
핵심 아이디어: Logistic Regression은 Linear Classification과 Linear Regression의 중간 — 실수 출력(like regression)이지만 [0,1]로 제한(like classification). 불확실성을 확률로 표현한다.

02
Logistic Regression 소개
동기: 심장마비 예측 예시 (슬라이드 3–4)
  • 입력 x: 콜레스테롤 수치, 혈압, 나이, 몸무게 등
  • 심장마비를 확실히 예측하기는 어려움 → 발생 확률을 예측하는 것이 적절
  • 원하는 출력: 0과 1 사이를 연속적으로 변하는 y (y가 1에 가까울수록 심장마비 발생 가능성 높음)
Linear Regression vs Logistic Regression (슬라이드 6)
  • Linear Regression (a):
    일부 확률값이 음수(< 0) — 확률로 부적절
  • Logistic Regression (b):
    모든 확률값이 0과 1 사이 — 확률로 적절
  • binary event를 다루지만, logistic은 불확실성 허용 → 중간값이 불확실성 반영
핵심 차이: Logistic Regression은 출력을 확률 범위 [0,1]로 smooth하게 제한한다.
03
Logistic Function θ(s)
정의 (슬라이드 8)
θ(s) = eˢ / (1 + eˢ) = 1 / (1 + e⁻ˢ)
  • 정의역: −∞ < s < ∞
  • 출력 범위: (0, 1) — 확률로 해석 가능
  • 별명 "soft threshold": 하드 임계값의 부드러운 버전
  • 별명 "sigmoid": 모양이 납작한 'S'자
1 − θ(s) = θ(−s)

→ 이 성질 덕분에 오차함수를 θ(ywTx)로 단순화 가능 (슬라이드 9)

왜 θ(s) = eˢ/(1+eˢ) 형태로 정의하는가?
이 형태 덕분에 1 − θ(s) = θ(−s) 가 성립하고, P(y|x) = θ(ywTx)로 깔끔하게 표현되어 오차 함수를 편리하게 최소화할 수 있다.
04
Hyperbolic Tangent & 함수 비교
tanh(s) — 또 다른 시그모이드 (슬라이드 12)
tanh(s) = (eˢ − e⁻ˢ) / (eˢ + e⁻ˢ)
  • 출력 범위: (−1, +1)
  • |s|가 크면: 하드 임계값에 수렴 (±1)
  • |s|가 작으면: 임계값 없이 거의 선형
  • 신경망(Neural Network)에서 활성화 함수로 자주 사용
4가지 함수 비교 그래프 (슬라이드 13)

logistic θ(s)는 [0,1], tanh는 [−1,+1], linear는 unbounded, hard threshold는 {−1,+1}

05
Big Picture — 확률 모델링 구조
학습 목표 (슬라이드 15)
  • 학습 목표 f(x) = ℙ[y = +1 | x] : 입력 x가 주어졌을 때 +1 사건이 발생할 확률
  • 예: 환자 정보 x가 주어졌을 때 심장마비(y=+1) 확률
  • 훈련 데이터는 f(x)를 직접 제공하지 않음 → 이 확률에 의해 생성된 샘플만 제공
이상적 데이터 (probability)
(x₁, f₁ = 0.99 = ℙ[+1|x₁])
(x₂, f₂ = 0.13 = ℙ[+1|x₂])
⋮
(xN, fN = 0.51 = ℙ[+1|xN])
실제 데이터 (binary event)
(x₁, y₁ = +1)
(x₂, y₂ = −1)
⋮
(xN, yN = −1)
확률 모델 공식 (슬라이드 14, 19)
ℙ[y = +1 | x] = f(x) ~ h(x) = θ(wTx)
P(y|x) = h(x)[y=+1] · (1−h(x))[y=−1] = θ(ywTx)
  • y=+1이면: P = h(x) = θ(wTx)
  • y=−1이면: P = 1−h(x) = θ(−wTx) (1−θ(s)=θ(−s) 이용)
  • 두 경우를 하나의 식 θ(ywTx)로 통합 — 이것이 logistic function 정의의 핵심 이유
06
오차 측정: Cross-Entropy Error
단순 오차 vs 크로스 엔트로피 (슬라이드 16–17)
오차 유형 수식 특징
단순 오차 Ei (1/N) Σ |h(xₙ) − ½(1+yₙ)|² 직관적이나 최소화 어려움
Cross-Entropy (1/N) Σ ln(1 + e^(−yₙwᵀxₙ)) 확률 해석 + 미분 최적화 유리
Cross-Entropy 오차의 의미 (슬라이드 18)
Eᵢₙ(w) = (1/N) Σ ln(1 + e^(−yₙwᵀxₙ))
  • yₙ = +1일 때: wᵀxₙ ≫ 0이면 오차↓ → θ(wᵀxₙ) ≈ 1 ✓
  • yₙ = −1일 때: wᵀxₙ ≪ 0이면 오차↓ → θ(wᵀxₙ) ≈ 0 ✓
  • 오차가 작으려면 signal이 레이블과 같은 부호여야 함
Cross Entropy 정의 (슬라이드 25)
두 확률분포 {p, 1−p}와 {q, 1−q}에 대해:
p·log(1/q) + (1−p)·log(1/(1−q))
→ 관측 분포를 피팅 분포로 근사하는 오차
07
Maximum Likelihood 유도
최대 우도 추정 (슬라이드 20–23)
  • 훈련 데이터 (x₁,y₁), …, (xN,yN) 은 독립적으로 생성되었다고 가정
  • 전체 데이터의 우도(Likelihood): ∏ P(yₙ|xₙ)
  • 최대 우도법: 이 확률을 최대화하는 w 선택
max ∏ P(yₙ|xₙ) ← maximize likelihood
⇔ max ln(∏ P(yₙ|xₙ)) ← log monotone 변환
≡ max Σ ln P(yₙ|xₙ) ← log product = sum
⇔ min −(1/N) Σ ln P(yₙ|xₙ) ← NLL (부호 반전)
≡ min (1/N) Σ ln(1 / θ(yₙwᵀxₙ)) ← P(y|x) = θ(ywᵀx) 대입
≡ min (1/N) Σ ln(1 + e^(−yₙwᵀxₙ)) ← 1/θ(s) = 1 + e^(-s)
= Eᵢₙ(w)  ← Cross-Entropy Error!
Cross-Entropy Loss 유도 요약 (슬라이드 25–28)
  • n번째 항의 cross-entropy:
    관측 pmf: {[yₙ=+1], [yₙ=−1]}
    피팅 pmf: {h(xₙ), 1−h(xₙ)}
  • Cross-entropy formula:
    [yₙ=+1]·ln(1/h(xₙ)) + [yₙ=−1]·ln(1/(1−h(xₙ)))
  • 이것이 ln(1 + e^(−yₙwᵀxₙ))와 같음을 보일 수 있다
결론: 3가지 관점이 동치
최소화 Eᵢₙ(w)
= 최소화 NLL(w)
= 최대화 Likelihood ∏P(yₙ|xₙ)
Eᵢₙ(w) = (1/N) Σ ln(1 + e^(−yₙwᵀxₙ))
  • 오차가 작으려면 yₙwᵀxₙ이 크고 양수여야 함
  • 즉 sign(wᵀx) = yₙ → 분류 정확
  • 오차함수가 각 xₙ을 올바르게 "분류"하도록 w를 유도
08
요약 — 핵심 정리
🔵 Logistic Regression

h(x) = θ(wTx). 출력이 (0,1)로 제한된 확률값. Binary event를 불확실성을 허용하며 예측. Linear와 Hard threshold의 중간.

🟣 Logistic Function θ

θ(s) = 1/(1+e-s). 핵심 성질: 1−θ(s) = θ(−s). 이 덕분에 P(y|x) = θ(ywTx)로 통합 표현 가능.

🟢 Cross-Entropy Error

Eᵢₙ(w) = (1/N)Σln(1+e-yₙwᵀxₙ). 최대 우도 추정(MLE)과 동치. 확률론적 해석 + gradient 최적화에 유리.

전체 학습 흐름
1 학습 목표: f(x) = ℙ[y=+1|x] 추정
↓
2 모델: h(x) = θ(wTx)로 근사
↓
3 오차: Cross-Entropy Eᵢₙ(w) = (1/N)Σln(1+e-yₙwᵀxₙ)
↓
4 최적화: Eᵢₙ(w) 최소화 ⟺ Likelihood 최대화
↓
5 결과: 각 xₙ을 올바르게 분류하는 w 획득
