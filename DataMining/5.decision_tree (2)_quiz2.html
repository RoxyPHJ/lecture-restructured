CART의 비용 함수 Rα(T) = R(T) + α·|T| 에서 α가 커질수록 어떤 트리가 선택되는가?
A
리프 노드가 많은 복잡한 트리
B
리프 노드가 적은 단순한 트리
C
훈련 오류가 가장 낮은 트리
D
α 값은 트리 구조에 영향을 주지 않는다.
정답 & 해설
정답: B
α는 복잡도 패널티 파라미터로, α가 클수록 |T|(리프 노드 수)에 대한 페널티가 커진다. 따라서 더 작고 단순한 트리(리프 노드가 적은)가 더 낮은 비용을 가져 선택된다. α=0이면 패널티가 없어 완전한 트리가 선택된다.

다음 중 Fβ-score 공식으로 올바른 것은?
A
2 × precision × recall / (precision + recall)
B
(1+β²) × precision × recall / (β² × precision + recall)
C
β × precision × recall / (precision + recall)
D
(1+β) × precision × recall / (β × precision + recall)
정답 & 해설
정답: B
Fβ = (1+β²)×precision×recall / (β²×precision+recall). β는 Recall에 대한 가중치를 조절하는 파라미터이다. β=1이면 F₁-score(Precision과 Recall 동일 가중치)가 된다. A는 F₁-score(β=1일 때)의 공식이다.

AdaBoost에서 올바르게 분류된 샘플의 가중치는 어떻게 변하는가?
A
증가한다 — 다음 모델이 이 샘플에 더 집중하도록
B
변하지 않는다 — 모든 샘플은 동일한 가중치를 유지

감소한다 — 상대적으로 어려운 샘플에 더 집중하도록

📄 Slide 15–16

A
wᵢ = error(Mᵢ) / (1 − error(Mᵢ))
B
wᵢ = log( (1 − error(Mᵢ)) / error(Mᵢ) )
C
wᵢ = 1 / error(Mᵢ)
D
wᵢ = 1 − error(Mᵢ)

정답: B

Bagging, Boosting, Random Forest를 올바르게 비교한 것은?

📄 Slide 4
CART의 비용 함수 Rα(T) = R(T) + α·|T| 에서 |T|가 의미하는 것은?
정답 & 해설
정답: 리프 노드의 수 (Number of Leaf Nodes)

📄 Slide 6
Accuracy(정확도) 공식을 완성하시오: Accuracy = ( TP + TN ) / ( ___ )
정답 & 해설
정답: P + N
Accuracy = (TP+TN)/(P+N). P는 실제 양성의 수(TP+FN), N은 실제 음성의 수(TN+FP)이며, P+N은 전체 샘플 수이다.

Precision(정밀도) 공식에서 분자는 무엇인가?
정답 & 해설
정답: TP (True Positive)
Precision = TP / (TP + FP). 분자는 TP(실제 양성을 양성으로 올바르게 예측한 수)이다. Precision은 "양성이라고 예측한 것 중 실제로 양성인 비율"로, 양성 예측의 신뢰성을 나타낸다.

📄 Slide 2–3
결정 트리에서 가지치기(Pruning)가 필요한 이유를 설명하고, Pre-Pruning과 Post-Pruning의 차이를 시점·방법·장단점 측면에서 비교하시오.
모범 답안
필요 이유
완전히 성장한 트리는 훈련 데이터의 노이즈·이상치까지 암기(memorize)하여 과적합되고, 미학습 데이터에서 성능이 저하된다. 가지치기는 트리를 단순화하여 일반화 성능을 향상시킨다.

Pre-Pruning | 시점: 트리 생성 도중 | 방법: 분할 기준(IG, Gini)이 유의미하지 않으면 노드를 리프로 전환 | 장점: 빠르고 간단 | 단점: 너무 일찍 중단 시 과소적합(Underfitting) 위험

Post-Pruning | 시점: 완전한 트리 생성 후 | 방법: 불필요한 서브트리를 리프(최빈 클래스)로 교체 | 장점: 더 신뢰할 수 있는 결과 | 단점: 계산 비용이 높음

공통 목표: 모델 복잡성과 일반화 능력의 균형

⭐⭐
📄 Slide 6
Confusion Matrix의 4가지 요소(TP, TN, FP, FN)를 각각 정의하고, 이를 기반으로 Accuracy, Precision, Recall(Sensitivity) 공식을 유도하시오.
모범 답안
Confusion Matrix 4요소
· TP(True Positive): 실제 양성 → 양성으로 올바르게 예측
· TN(True Negative): 실제 음성 → 음성으로 올바르게 예측
· FP(False Positive): 실제 음성 → 양성으로 잘못 예측 (오경보, False Alarm)
· FN(False Negative): 실제 양성 → 음성으로 잘못 예측 (누락, Miss)

공식
· Accuracy = (TP+TN) / (P+N) — 전체 중 올바른 예측 비율
· Precision = TP / (TP+FP) — 양성 예측 중 실제 양성 비율
· Recall(Sensitivity) = TP / P = TP / (TP+FN) — 실제 양성 중 탐지된 비율

참고: Accuracy는 클래스 균형 시 적합, 불균형 시 Precision·Recall 사용 권장

⭐⭐
📄 Slide 7–8
클래스 불균형(Class Imbalance) 문제를 구체적인 예시와 함께 설명하고, 이를 해결하기 위한 평가 지표(F₁-score 포함)와 각 지표가 중요한 이유를 서술하시오.
모범 답안
문제 설명
실제 데이터(예: 사기 탐지에서 정상 거래 99%, 사기 1%)에서 다수 클래스만 예측해도 높은 Accuracy가 나온다. → 97% 음성 데이터에서 모두 음성으로 예측해도 Accuracy=97%지만 Recall=0%

해결 지표
· Precision = TP/(TP+FP): 양성 예측의 신뢰성 → FP(오경보)를 줄이는 데 중요
· Recall = TP/P: 실제 양성 탐지 능력 → FN(누락)을 줄이는 데 중요. 의료·보안에서 치명적
· F₁-score = 2×P×R/(P+R): Precision과 Recall의 조화 평균 → 두 지표 균형 평가

결론: 중요도에 따라 FP와 FN 중 어느 것을 더 줄여야 하는지 파악 후 적절한 지표 선택

주관식
⭐⭐⭐
📄 Slide 11–16
Bagging과 Boosting의 작동 원리를 비교 설명하고, 각각이 주로 감소시키는 오류의 종류(분산 vs 편향)와 그 이유를 서술하시오.
모범 답안
Bagging
· 원리: Bootstrap Sampling(복원 추출)으로 D₁~Dₖ 생성 → 병렬(Parallel)로 각 Mᵢ 훈련 → 다수결/평균으로 최종 예측
· 주로 감소: 분산(Variance) — 여러 모델을 평균하면 개별 모델의 변동성이 상쇄됨
· 예: 하나의 모델이 노이즈에 민감해도 전체 평균은 안정적

Boosting (AdaBoost)
· 원리: 동일 가중치 초기화 → 순차(Sequential) 학습 → 잘못 분류 샘플 가중치 증가 → 다음 모델이 어려운 샘플에 집중 → 가중 투표로 최종 예측
· 주로 감소: 편향(Bias) — 어려운 샘플에 반복 집중하여 복잡한 패턴도 학습 가능
· 주의: 노이즈가 많은 데이터에서는 과적합 위험이 있음

핵심 차이: 병렬 vs 순차, 분산 감소 vs 편향 감소

📄 Slide 18–19
Random Forest의 훈련 과정과 예측 과정을 단계별로 서술하고, 단일 결정 트리 대비 우수한 이유를 두 가지 핵심 기법과 연결하여 설명하시오.
모범 답안
훈련 과정
① Bootstrap Sampling으로 D에서 복원 추출하여 Dᵢ 생성
② 각 Dᵢ로 결정 트리 성장(grow)
③ 각 분기(split) 시 전체 특성 중 무작위로 F개만 선택
④ 선택된 F개 특성 중 최적 분기 결정
⑤ k개의 서로 다른 트리 M₁, M₂, …, Mₖ 완성

예측 과정
새 샘플 X → 각 트리가 독립적으로 예측 → 분류: 다수결 투표, 회귀: 평균

우수한 이유
① Bagging: Bootstrap으로 다양한 훈련 데이터 → 분산(Variance)↓
② 무작위 특성 선택: 강한 특성에 의한 트리 간 상관관계 억제 → 다양성↑, 분산↓
→ 단일 트리의 불안정성(높은 분산)을 두 기법이 동시에 해결하여 더 안정적이고 정확한 예측 가능

📄 Slide 4–19 종합
Decision Tree (2) 강의 전체를 종합하여, "가지치기 → 모델 평가 → 앙상블" 의 흐름이 왜 자연스럽게 연결되는지 각 개념의 필요성과 관계를 중심으로 서술하시오.
모범 답안
가지치기 → 모델 평가
단일 결정 트리는 과적합 위험이 있으므로 가지치기로 복잡성을 줄인다. 하지만 가지치기 후에도 모델이 실제로 일반화 성능이 향상되었는지 검증해야 한다. 훈련 정확도만 보면 과적합을 감지할 수 없으므로, Holdout·Cross-Validation·Bootstrap 등 신뢰할 수 있는 평가 방법이 필요하다.

모델 평가 → 앙상블
단일 모델을 아무리 잘 평가하고 가지치기해도 근본적인 한계(단일 모델의 분산·편향)가 남는다. 앙상블은 이 한계를 뛰어넘기 위해 여러 모델을 결합한다. Bagging은 분산을 줄이고, Boosting은 편향을 줄이며, Random Forest는 두 기법을 결합한다.

종합
과적합 문제(가지치기) → 평가의 신뢰성 확보(모델 평가) → 단일 모델의 한계 극복(앙상블)의 흐름은 머신러닝에서 더 나은 일반화 성능을 추구하는 자연스러운 발전 방향이다.
