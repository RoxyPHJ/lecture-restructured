01 클러스터 분석이란 (What is Cluster Analysis) Slide 2

클러스터 분석은 데이터 객체 집합을 클러스터(cluster)라 불리는 부분집합으로 분할하는 작업이다. 같은 클러스터 내 객체는 서로 유사하고, 다른 클러스터의 객체와는 비유사하다.

핵심 특성
  • 비지도 학습(Unsupervised Learning): 클래스 레이블 없이 데이터로부터 그룹을 자동 발견
  • 동일 데이터에서도 방법·파라미터·초기화에 따라 다른 클러스터링 결과가 나올 수 있음
목적
  • 데이터에서 이전에 알려지지 않은 그룹·구조를 발견
  • 각 클러스터 → 암묵적 클래스(implicit class)로 취급 → 이후 사람이 해석·레이블링
지도학습 vs 비지도학습 (클러스터링)

03 요구사항 #1 ~ #2 (Requirements for Cluster Analysis) Slide 4

실용적인 클러스터링 알고리즘은 단순한 거리 기반 그룹화를 넘어 여러 요구사항을 충족해야 한다. 실세계 데이터는 대규모, 노이즈·불완전, 고차원, 이질적인 특성을 가진다.

#1
확장성 (Scalability)

현대 DB·웹 규모 앱은 수백만~수십억 객체 보유. 소표본 클러스터링은 편향·불안정 결과를 초래. 시간·메모리 복잡도가 데이터 크기 N에 대해 합리적으로(예: O(N)에 가깝게) 스케일되어야 함

#2
임의 형태 클러스터 발견

거리 기반 방법은 구형(spherical) 클러스터 선호. 실제로는 임의의 비볼록(non-convex) 불규칙 클러스터가 존재. 이를 발견할 수 있는 알고리즘 필요

시간/메모리 복잡도 비교 — Scalability
04 요구사항 #3 ~ #4 Slide 5
#3
노이즈·이상치에 대한 강인성

실세계 데이터는 측정 오류, 결측값, 이상치 포함. 적당량의 노이즈를 허용하거나, 이상치를 명시적으로 식별하거나 그 영향을 줄여야 함

#4
고차원 데이터 처리

현대 데이터의 고차원 → 차원의 저주(Curse of Dimensionality)
거리가 덜 의미있어지고, 데이터가 희소·편향됨
대응책: 차원 축소, 특성 선택, 투영 기반 클러스터링

차원의 저주 — 핵심 포인트
거리가 의미 없어짐
데이터가 희소·편향
UMAP / PCA / Autoencoder
Feature Selection
Subspace / Manifold Learning
05 요구사항 #5 ~ #6 Slide 6
#5
도메인 지식 의존도 최소화

많은 방법이 클러스터 수 k나 거리·밀도 임계값 같은 파라미터를 사용자 설정 필요 → 고차원에서 선택이 매우 어렵고 민감함. 데이터 기반(data-driven)으로 더 적고 해석 가능한 파라미터가 선호됨

#6
해석 가능성 & 사용성

클러스터 결과가 도메인 전문가와 의사결정자가 이해할 수 있어야 함. 각 클러스터가 무엇을 나타내는지 명확한 의미 개념과 연결 가능해야 함. 특성 선택·데이터 표현·알고리즘 선택은 최종 응용을 고려

06 기본 클러스터링 방법론 개요 Slide 7–10
Partitioning
분할 방법 (Partitioning Methods)
n개 객체를 k개 그룹으로 분할. 각 객체는 정확히 하나의 클러스터 소속 (퍼지 분할에서는 확률적 소속 가능). 반복적으로 객체를 재배치하여 클러스터 품질 개선. 대표 알고리즘: k-means, k-medoids
✓ 구형 클러스터에 효과적, 대용량 가능
✗ k 사전 지정, 이상치에 민감, 비구형 클러스터 약점
Hierarchical
계층적 방법 (Hierarchical Methods)
단일 평면 분할 대신 계층적 분해 구축.
• 응집형(Agglomerative, 상향식): 각 객체를 개별 클러스터로 시작 → 가까운 클러스터 반복 병합
• 분열형(Divisive, 하향식): 전체를 하나로 시작 → 반복 분할
✓ 덴드로그램으로 시각화, k 사전 지정 불필요
✗ 한번 병합/분할은 되돌리기 불가
Density-based
밀도 기반 방법 (Density-based Methods)
클러스터를 지역 밀도가 충분히 높은 영역으로 정의. 지정 반경 내 최소 포인트 수를 만족하면 클러스터를 확장. 대표 알고리즘: DBSCAN
✓ 임의 형태 발견 가능, 노이즈·이상치 자연스럽게 식별
✗ 밀도 임계값 파라미터 설정 필요, 고차원에서 약점
Grid-based
격자 기반 방법 (Grid-based Methods)
데이터 공간을 유한 개의 셀(격자)로 양자화 → 모든 객체 대신 격자 위에서 클러스터링. 밀집 셀을 클러스터 구성요소로, 인접 밀집 셀을 결합하여 더 큰 클러스터 구성
✓ 높은 효율성 — 처리 비용은 셀 수에 의존 (데이터 수 무관)
✗ 셀 크기(해상도) 선택에 민감
Grid-based Method: 3단계 프로세스
07 Partitioning Methods & k-means 직관 Slide 11–12
문제 설정 (Problem Setup)
  • 입력: n개 객체를 포함하는 데이터 D, 원하는 클러스터 수 k (k ≤ n)
  • 목표: D를 k개의 클러스터 C₁, C₂, …, C_k로 분할 (각 클러스터 비공, 상호 배타적)
  • 최적화 기준: intra-cluster similarity 최대화 + inter-cluster similarity 최소화
k-means: Centroid 기반 분할

각 클러스터 C_i는 센트로이드(centroid) c_i를 가짐 — C_i에 속한 포인트들의 평균값으로 정의 (유클리드 공간)

E = Σ(i=1 to k) Σ(p∈C_i) dist(p, c_i)²
SSE (Within-Cluster Sum of Squared Errors) — 이 값을 최소화하는 분할을 찾는 것이 목표
dist: 보통 유클리드 거리

왜 휴리스틱이 필요한가? SSE 정확 최소화는 NP-hard (k=2에서도). 가능한 분할 수가 n에 대해 지수적으로 증가하기 때문. 따라서 실용적 알고리즘은 탐욕적/반복적 휴리스틱으로 지역 최적해(local optimum)에 근접

08 k-means 알고리즘: 절차 & 시간 복잡도 Slide 13, 15
기본 절차 (Basic Procedure)
1
데이터 D에서 k개 객체를 무작위로 선택하여 초기 센트로이드로 설정
2
할당 단계(Assignment Step): 각 객체를 가장 가까운 센트로이드의 클러스터에 할당
3
갱신 단계(Update Step): 각 클러스터의 센트로이드를 소속 객체들의 평균으로 재계산
4
클러스터 할당이 더 이상 변하지 않을 때까지(수렴) 2~3단계 반복
시간 복잡도
O(n · k · t)
n: 객체 수 | k: 클러스터 수 | t: 수렴까지 반복 횟수 (보통 t ≪ n)

n에 대해 선형 스케일. 중간 규모 차원의 대용량 데이터셋에서 잘 작동

k-means 수렴 과정 (개념도)
09 k-means 장점 & 단점 Slide 14
✓ 장점 (Advantages)
  • 개념적으로 직관적이고 구현 쉬움
  • 통계, 데이터 마이닝, ML 툴킷에 광범위하게 제공
  • 어떤 지역 최적해로 수렴 보장
  • Warm Start 지원 (사용자 지정 초기 센트로이드)
✗ 단점 (Limitations)
  • k를 사전에 지정해야 함
  • 초기 센트로이드에 민감: 다양한 초기화로 여러 번 실행 필요
  • 크기·밀도가 매우 다른 클러스터에 취약, 이상치에 민감
  • 고차원 데이터에서 성능 저하

10 k-means 이상치 민감성 예시 (Drawback) Slide 16
예시 데이터 (1D)
D = {1, 2, 3, 8, 9, 10, 25}

직관적으로 {1,2,3}과 {8,9,10}이 두 클러스터, 25는 이상치 — 하지만 k-means는?

k-means (k=2) SSE 비교
A { {1,2,3}, {8,9,10,25} } → 평균: 2, 13 SSE = 196
B { {1,2,3,8}, {9,10,25} } → 평균: 3.5, 14.67 SSE ≈ 189.67 ← k-means 선택

k-means는 SSE가 더 낮은 분할 B를 선택하여 8을 9, 10과 분리시킴. 이상치 25의 영향으로 두 번째 센트로이드(14.67)가 실제 어떤 데이터와도 멀리 떨어진 나쁜 대표값이 됨

분할 A vs 분할 B — 1D 시각화

핵심 단점: k-means는 이상치에 매우 민감하다. 극단값 몇 개가 클러스터 할당을 왜곡하고, 실제 데이터 포인트와 대응되지 않는 센트로이드를 만들어낸다. → 이를 보완하기 위해 k-medoids 같은 변형 알고리즘이 제안됨
