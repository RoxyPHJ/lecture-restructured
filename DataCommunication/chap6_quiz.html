📡 FDM — 주파수 분할 다중화
Q1 객관식 FDM ⭐ Slide 5
FDM(주파수 분할 다중화)에 대한 설명으로 옳은 것은?
  • A디지털 신호를 시간 슬롯으로 나누어 전송하는 기술이다.
  • B링크의 대역폭이 전송되는 조합 신호의 대역폭보다 클 때 적용 가능한 아날로그 기술이다.
  • C채널 간 간섭 방지를 위해 반드시 동일한 반송 주파수를 사용해야 한다.
  • D전송 매체의 데이터 전송률이 채널 요구 전송률보다 낮을 때 사용한다.
정답 — B
FDM은 아날로그 기술로, 링크 대역폭 > 조합 신호 대역폭일 때 적용합니다.
각 채널은 서로 다른 반송 주파수(f₁, f₂, f₃)로 변조되며, 채널 간 보호대역(guard band)으로 간섭을 방지합니다.
A·D는 TDM에 대한 설명이고, C는 틀린 설명입니다.
Q2 객관식 [응용] FDM 응용 ⭐⭐ Slide 9
FM 라디오 방송은 87.5 MHz ~ 108 MHz 대역을 사용하고, 방송국당 대역폭은 200 kHz이다. 이 대역에서 이론상 최대 몇 개의 방송국이 동시에 운영 가능한가? (보호대역은 무시)
  • A약 87개
  • B약 100개
  • C약 104개
  • D약 540개
정답 — C
[응용 풀이] FDM에서 채널 수 = 전체 대역폭 / 채널당 대역폭
전체 대역폭 = 108 MHz - 87.5 MHz = 20.5 MHz = 20,500 kHz 채널 수 = 20,500 kHz ÷ 200 kHz = 102.5 → 약 102~104개
적용 개념: FDM은 대역폭을 주파수로 분할하며, 각 채널이 동일한 대역폭을 점유합니다.
Q3 OX FDM ⭐ Slide 5-7
FDM에서 보호대역(Guard Band)은 인접 채널 신호가 서로 겹치는 것을 방지하기 위해 채널 사이에 삽입되는 미사용 주파수 구간이다.
  • O옳다
  • X틀리다
정답 — O (옳다)
보호대역은 FDM의 핵심 개념으로, 각 채널을 서로 다른 반송파로 변조할 때 인접 채널 간 간섭(interference)을 막기 위한 미사용 주파수 간격입니다.
이로 인해 링크의 실제 사용 가능한 채널 수가 줄어드는 단점이 있습니다.
⏱️ TDM — 시분할 다중화
Q4 객관식 TDM ⭐ Slide 10-12
TDM(시분할 다중화)에서 끼워넣기(Interleaving)에 대한 설명으로 옳은 것은?
  • A각 채널에 전송할 데이터가 있을 때만 슬롯을 동적으로 할당한다.
  • B스위치가 채널들을 일정한 비율로 정해진 순서대로 순환하며 각 채널에 슬롯을 할당한다.
  • C주파수 대역을 나누어 각 채널에 고정 주파수를 할당하는 방식이다.
  • D수신 측에서 의사난수 코드로 원래 신호를 복원하는 방식이다.
정답 — B
TDM의 끼워넣기는 MUX의 스위치가 채널들을 고정된 순서로 순환하며 각 채널에서 1 슬롯씩 가져와 프레임을 구성하는 방식입니다.
A는 통계적 TDM, C는 FDM, D는 DSSS에 대한 설명입니다.
Q5 계산형 TDM ⭐⭐ Slide 13 — Example 6.8
4개 채널을 TDM으로 다중화한다. 각 채널은 100 bytes/s를 전송하고, 1 byte씩 인터리빙할 때, 링크의 비트율(bps)을 구하시오.
정답 — 3200 bps
프레임 크기 = 4채널 × 1 byte = 4 bytes = 32 bits 프레임 률 = 100 frames/s (채널당 100 bytes/s, 1 byte/frame) 링크 비트율 = 100 × 32 = 3,200 bps
강의 Example 6.8의 직접 계산 문제입니다.
Q6 계산형 [응용] TDM ⭐⭐⭐ Slide 13 — 수치 변형
6개 채널을 TDM으로 다중화한다. 각 채널은 200 bytes/s를 전송하고, 2 bytes씩 인터리빙할 때, 다음을 구하시오.

① 프레임 크기 (bytes)   ② 프레임률 (frames/s)   ③ 링크 비트율 (bps)
정답
① 프레임 크기 = 6채널 × 2 bytes = 12 bytes = 96 bits ② 프레임 률 = 200 bytes/s ÷ 2 bytes/frame = 100 frames/s ③ 링크 비트율 = 100 × 96 = 9,600 bps
[응용 적용 개념]
Example 6.8과 동일한 공식 구조이지만 채널 수(4→6), 전송률(100→200 bytes/s), 슬롯 크기(1→2 bytes)가 변형되었습니다.
핵심: 프레임률 = 채널 전송률 ÷ 슬롯 크기 (단위 맞추기 주의)
Q7 OX [응용] TDM — T회선 ⭐⭐ Slide 17-20
T-1 회선은 DS-1 신호를 전송하며, 하나의 T-1 프레임은 24개 채널 × 8 bits + 1 framing bit = 193 bits로 구성된다. 프레임률이 8,000 frames/s이므로 T-1의 데이터 전송률은 1.544 Mbps이다.
  • O옳다
  • X틀리다
정답 — O (옳다)
프레임 크기 = 24 × 8 + 1 = 193 bits 전송률 = 8,000 × 193 = 1,544,000 bps = 1.544 Mbps
[응용 포인트] 단순 암기가 아닌, 프레임 구조에서 전송률을 도출하는 과정을 이해해야 합니다.
8,000 frames/s는 전화 음성의 Nyquist 샘플링 주파수(8 kHz)에서 기인합니다.
Q8 단답형 TDM — DS계층 ⭐ Slide 17-18
DS-1 신호를 전송하는 회선명은 무엇이며, 그 전송 속도는 얼마인가? (형식: 회선명 / 전송속도)
정답 — T-1 / 1.544 Mbps
DS 계층 구조에서 DS-1 ↔ T-1 (1.544 Mbps, 24채널)이 대응됩니다.
유럽·국내에서는 동등 수준으로 E-1 (2.048 Mbps, 30채널)을 사용합니다.
Q9 종합판단형 TDM ⭐⭐ Slide 10-22
TDM에 관한 다음 설명 중 틀린 것은?
  • ATDM은 전송 매체의 데이터 전송률이 채널 요구 전송률보다 클 때 적용하는 디지털 기술이다.
  • B동기식 TDM에서 발신자가 전송할 데이터가 없으면 해당 슬롯은 비어 있게 된다.
  • C프레임 구성비트(Framing bits)는 수신 측에서 프레임 경계를 인식하는 데 사용된다.
  • DT-4 회선은 T-1 회선 4개를 묶어 만들며, 전송속도는 약 6.312 Mbps이다.
정답 — D
T-4는 T-1 168개를 묶으며 전송속도는 274.176 Mbps입니다.
T-1 4개를 묶은 것은 T-2 (6.312 Mbps)입니다.
A·B·C는 모두 강의 내용에 부합하는 정확한 설명입니다.
📊 통계적 TDM
Q10 빈칸추론형 통계적 TDM ⭐⭐ Slide 22
다음 문장의 (A)와 (B)에 들어갈 올바른 단어 조합을 고르시오.

"동기식 TDM은 발신자에게 데이터가 없어도 고정 슬롯이 할당되므로 (A) 이/가 발생한다. 반면 통계적 TDM은 데이터가 있는 채널에만 슬롯을 (B) 으로/로 할당하므로 링크 효율이 높아진다."
  • A(A) 보호대역 / (B) 정적
  • B(A) 빈 슬롯 / (B) 동적
  • C(A) 프레임 오류 / (B) 정적
  • D(A) 빈 슬롯 / (B) 정적
정답 — B
동기식 TDM의 핵심 단점은 빈 슬롯(Empty Slot) 낭비입니다.
통계적 TDM은 동적(Dynamic) 슬롯 할당으로 이를 해결하지만, 각 슬롯에 수신지 주소 정보를 포함해야 하는 오버헤드가 발생합니다.
Q11 객관식 [응용] 통계적 TDM ⭐⭐⭐ Slide 22
통계적 TDM에 대한 설명으로 옳지 않은 것은?
  • A동기식 TDM보다 링크 대역폭을 효율적으로 사용할 수 있다.
  • B슬롯에 주소 정보를 포함해야 하므로 오버헤드가 발생한다.
  • C연결 가능한 채널 수가 링크의 슬롯 수보다 많을 수 있다.
  • D모든 채널이 동시에 최대 부하로 전송할 때 지연 없이 처리된다.
정답 — D
[응용 포인트] 통계적 TDM은 평균 부하를 기반으로 설계합니다.
모든 채널이 동시에 최대 부하를 발생시키면 슬롯이 부족하여 지연(Delay) 또는 손실이 발생할 수 있습니다.
이것이 통계적 TDM의 근본적인 한계입니다. A·B·C는 모두 올바른 설명입니다.
Q12 OX 통계적 TDM ⭐ Slide 22
통계적 TDM에서 각 슬롯에는 데이터와 함께 수신지 주소 정보가 포함되어야 하므로, 동기식 TDM에 비해 슬롯당 오버헤드가 더 크다.
  • O옳다
  • X틀리다
정답 — O (옳다)
동기식 TDM은 슬롯 위치 자체가 채널을 식별하므로 주소 정보가 불필요합니다.
반면 통계적 TDM은 슬롯이 동적으로 할당되기 때문에 수신 측이 어느 채널의 데이터인지 알 수 있도록 주소(Address) 정보를 슬롯에 포함해야 합니다.
📡 확산 대역 방식 (FHSS / DSSS)
Q13 객관식 FHSS ⭐ Slide 24-27
FHSS(주파수 뛰기 대역 확산)의 동작 방식으로 옳은 것은?
  • A데이터 비트를 n개의 칩(chip)으로 확산하여 대역폭을 넓힌다.
  • B의사난수 코드에 의해 결정된 순서로 주파수를 이동하며 신호를 전송한다.
  • C고정된 단일 주파수 채널에서 시간 슬롯을 나누어 전송한다.
  • D보호대역을 사용하여 채널 간 간섭을 방지한다.
정답 — B
FHSS는 의사난수(Pseudorandom) 코드가 정한 순서에 따라 반송파 주파수를 주기적으로 바꾸며 전송합니다.
A는 DSSS, C는 TDM, D는 FDM에 대한 설명입니다.
송수신 양측이 동일한 의사난수 코드를 공유해야 동기화됩니다.
Q14 단답형 DSSS ⭐⭐ Slide 28
DSSS에서 각 데이터 비트를 n개의 비트로 대체하는 데 사용하는 코드의 명칭을 쓰시오.
정답 — 확산 코드 (Spreading Code)
DSSS에서 데이터 비트 "1"은 확산 코드 그대로, "0"은 확산 코드의 보수(complement)로 대체합니다.
확산 코드의 길이 n에 비례하여 대역폭이 n배 증가하며, 이 비율을 처리 이득(Processing Gain)이라 합니다.
Q15 종합판단형 [응용] FHSS vs DSSS ⭐⭐⭐ Slide 23-29
FHSS와 DSSS를 비교한 설명 중 옳지 않은 것은?
  • A두 방식 모두 확산 대역 방식으로, 무선 통신에서 간섭 및 도청에 강인하다.
  • BFHSS는 시간에 따라 반송 주파수를 바꾸며, DSSS는 각 비트를 코드로 확산하여 대역폭을 넓힌다.
  • CDSSS에서 확산 코드의 길이 n이 클수록 사용 대역폭이 좁아져 채널 효율이 증가한다.
  • DFHSS에서 여러 사용자는 서로 다른 의사난수 코드를 사용하여 동일 대역폭을 공유할 수 있다.
정답 — C
[응용 포인트] DSSS에서 확산 코드 길이 n이 클수록 사용 대역폭은 n배 증가합니다.
"대역폭이 좁아진다"는 정반대의 설명입니다.
대역폭이 넓어지면 채널 용량 측면에서 손실이 있지만, 보안성·잡음 내성이 향상됩니다.
이것이 확산 대역 방식의 핵심 트레이드오프입니다.
Q16 계산형 [응용] DSSS ⭐⭐⭐ Slide 28-29
DSSS 방식에서 원래 데이터 전송률이 1 Mbps이고, 확산 코드의 길이가 n = 11일 때, 전송에 필요한 대역폭(칩 전송률, chip rate)은 얼마인가?
정답 — 11 Mbps
칩 전송률 (Chip Rate) = 데이터 전송률 × n = 1 Mbps × 11 = 11 Mbps
[응용 적용 개념]
DSSS는 1비트를 n칩으로 대체하므로 전송에 필요한 대역폭이 n배 증가합니다.
참고: IEEE 802.11b Wi-Fi 표준이 11칩 Barker 코드를 사용하는 DSSS를 채택했습니다.

"(A) 은/는 링크 대역폭을 주파수 축으로 분할하여 아날로그 채널을 동시에 전송하며, (B) 은/는 링크 대역폭을 시간 축으로 분할하여 디지털 채널을 순차적으로 전송한다. 두 방식 모두 채널 수와 링크 용량 사이의 사전 계획된 고정 할당을 전제로 한다는 공통점이 있다."
  • A(A) FHSS / (B) DSSS

Q18 단답형 [응용] DS계층 역산 ⭐⭐⭐ Slide 17-18
DS 계층에서 T-3 회선은 DS-1 신호 몇 개를 묶은 것인가? (정수로 답할 것)
정답 — 28개
T-3 (DS-3) = 44.736 Mbps DS-1 (T-1) = 1.544 Mbps 비율 = 44.736 / 1.544 ≈ 28.97 → 정확히 DS-1 × 28
[응용 포인트] 단순 암기가 아닌 DS 계층 배수 관계를 이해해야 합니다.
DS-1 × 4 = DS-2 (T-2), DS-1 × 28 = DS-3 (T-3), DS-1 × 168 = DS-4 (T-4)

FDM은 아날로그 기술로, 링크의 대역폭이 전송되는 조합 신호의 대역폭보다 클 때 적용 가능하다. 각 채널을 서로 다른 반송 주파수로 변조하여 주파수 축으로 분할하며, 채널 간 간섭 방지를 위해 보호대역을 삽입한다.

TDM은 디지털 기술로, 전송 매체의 데이터 전송률이 채널 요구 전송률보다 클 때 적용 가능하다. 링크 사용 시간을 슬롯으로 나누고, 각 채널에 순서대로 슬롯을 할당하는 시간 축 분할 방식이다.

공통점은 두 방식 모두 단일 링크로 다수의 채널을 동시에 수용한다는 점이다.
⚠️ 주요 감점 요인
• "아날로그/디지털" 구분 미기재
• 적용 조건(대역폭 vs 전송률)을 혼동하거나 생략
• "주파수 분할" vs "시간 분할" 핵심어 누락
• 보호대역 개념 미언급
Q20 서술형 [응용] 확산 대역 방식 ⭐⭐⭐ Slide 23-29
확산 대역 방식(Spread Spectrum)이 왜 무선 통신에서 사용되는지, FHSS와 DSSS 각각의 원리와 장점을 포함하여 서술하시오.
📌 채점 기준 키워드
간섭 내성 도청 방지 의사난수 코드 주파수 뛰기 코드 확산 / 칩
모범 답안
무선 통신 환경은 간섭(Interference)과 도청(Eavesdropping)에 취약하다. 확산 대역 방식은 신호에 여분의 정보를 추가하여 대역폭을 넓힘으로써 이러한 문제에 강인하게 대응한다.

FHSS는 의사난수 코드가 결정하는 순서에 따라 반송파 주파수를 주기적으로 이동시킨다. 특정 주파수가 방해를 받더라도 전체 통신은 계속되며, 동일 대역폭에서 서로 다른 코드를 사용하는 다수 사용자가 공존할 수 있다.

DSSS는 각 데이터 비트를 확산 코드(길이 n)를 사용하여 n개의 칩으로 대체한다. 대역폭이 n배 넓어지고 신호 에너지가 분산되어 잡음 및 도청에 강인해진다. 수신 측은 동일한 확산 코드로 원래 비트를 복원한다.
⚠️ 주요 감점 요인
• 확산 대역 방식의 도입 목적(간섭·도청) 미기재
• FHSS와 DSSS를 구분하지 않고 혼재하여 서술
• "의사난수 코드"를 언급하지 않은 경우
• DSSS에서 "칩(chip)"과 대역폭 n배 확산 관계 미언급
