01
다중화 개요
단일 데이터 링크를 통해 여러 개의 신호를 동시에 전송하기 위한 기술 Slide 2
핵심 구성요소
  • MUX (Multiplexer) — 여러 전송 스트림 → 단일 스트림 결합 (many to one)
  • DEMUX (Demultiplexer) — 단일 스트림 → 각각의 요소로 분리 (one to many)
  • 링크 (Link) — 물리적인 경로
  • 채널 (Channel) — 한 쌍의 장치 간 전송을 위한 하나의 경로
다중화의 범주
  • FDM — 주파수 분할 다중화 (아날로그)
  • TDM — 시분할 다중화 (디지털)
  • FHSS — 주파수 뛰기 대역 확산 (무선)
  • DSSS — 직접 순열 확산 (무선)

02
FDM — 주파수 분할 다중화
아날로그
링크의 대역폭이 전송되는 조합 신호의 대역폭보다 클 때 적용할 수 있는 아날로그 기술 Slide 5
FDM 주파수 배치 구조
  • 각 채널은 서로 다른 반송 주파수(f₁, f₂, f₃)로 변조
  • 보호대역 (Guard Band) — 신호 겹침 방지
  • DEMUX에서 필터로 각 채널 분리
링크 대역폭 > 합산 채널 대역폭 각 채널 = 원신호 + guard band
변조 과정: 원신호 → 반송파(f₁)로 변조 → 합산(MUX) → 링크 → DEMUX 필터 → 복조

03
TDM — 시분할 다중화
디지털
송신·수신 장치가 요구하는 데이터 전송률보다 전송 매체의 전송률이 클 때 적용되는 디지털 처리 기술 Slide 10
TDM 프레임 구조 — 끼워넣기(Interleaving)
  • 시간 슬롯 (Time Slot) — 각 채널에 할당된 시간 단위
  • 프레임 (Frame) — 각 채널에서 1개씩 슬롯을 모은 단위
  • 끼워넣기 (Interleaving) — 스위치가 채널을 순환하며 슬롯 할당
  • 빈 슬롯 — 발신자가 데이터 없으면 슬롯 낭비 발생
프레임 크기 = 채널 수 × 슬롯 크기 프레임률 = 채널당 전송률 / 슬롯 크기 링크 비트률 = 프레임률 × 프레임 크기(bits)
▶ EXAMPLE 6.8 — TDM 계산 Slide 13
4개 채널을 TDM으로 다중화. 각 채널은 100 bytes/s, 1 byte씩 인터리빙
프레임 크기 = 4채널 × 1byte = 4 bytes = 32 bits 프레임 률 = 100 frames/s (채널당 100 bytes/s, 1byte/frame이므로) 링크 비트율 = 100 × 32 = 3,200 bps

04
통계적 TDM (Statistical TDM)
디지털
동기식 TDM vs 통계적 TDM 비교 Slide 22
동기식 TDM: 각 채널에 고정 슬롯 할당 → 데이터 없어도 빈 슬롯 낭비
통계적 TDM: 데이터가 있는 채널에만 동적으로 슬롯 할당 → 링크 효율 향상
  • 슬롯에 주소 정보를 포함해야 함 (오버헤드 발생)
  • 채널 수 > 슬롯 수 가능 (통계적 다중화 이득)
  • 부하가 클 때 지연 발생 가능
05
확산 대역 방식 (Spread Spectrum)
무선
무선 통신 응용을 위해 설계 — 신호에 여분의 정보를 추가하여 대역폭을 넓힘으로써 간섭·도청에 강인하게 만드는 기술 Slide 23
FHSS — 주파수 뛰기 대역 확산 Slide 24-27
  • M개의 신호를 서로 다른 반송파로 변조
  • 의사난수(Pseudorandom) 코드로 주파수 선택 순서 결정
  • 송신·수신 측이 동일한 코드를 공유해야 동기화
  • 넓은 대역폭 사용 → 간섭·도청에 강인
  • 대역폭 공유: 여러 사용자가 서로 다른 코드로 공존
특징: 주파수가 시간에 따라 랜덤하게 점프 → 특정 주파수 방해해도 전체 통신 영향 미미
DSSS — 직접 순열 확산 방식 Slide 28-29
  • 각 데이터 비트를 확산 코드(Spreading Code)로 n비트로 대체
  • 대역폭이 n배로 확산됨
  • 칩률(Chip Rate) = 데이터 비트율 × n
  • 수신 측에서 동일 코드로 XOR → 원래 데이터 복원
  • 동일 대역폭에 다수 사용자 공존 가능 (CDMA 기반)
데이터 비트 "1" → 확산 코드 (예: 10110010) 데이터 비트 "0" → 확산 코드 보수 (예: 01001101) 대역폭 이득 = n (확산 코드 길이)
